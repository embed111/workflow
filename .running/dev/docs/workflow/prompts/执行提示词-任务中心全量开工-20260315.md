## 2. 本次任务性质
1. 本次是 `任务中心` 第一版的正式实现开工，不是继续补静态示意图。
2. 当前交付已覆盖三部分：任务图与调度、产物交付、删除管理动作。
3. 必须按现有文档与原型实现，不要再自行发明旧版迁移语义或独立模块。

## 3. 本次必须完成的范围
1. 左侧导航接入 `任务中心`，并保持 5 个顶层模块：
   1. `对话`
   2. `工作记录与训练`
   3. `训练中心`
   4. `任务中心`
   5. `设置`
2. 页面主结构按原型实现：
   1. 页头调度状态与动作区
   2. 中部任务图
   3. 右侧任务详情
3. 页头必须实现：
   1. 状态：`运行中 / 暂停中 / 已暂停 / 未启动`
   2. 统计：`并行agent数`
   3. 动作：`暂停调度 / 继续运行 / 清空任务 / + 创建任务`
4. 创建任务必须实现为抽屉表单。
5. 当前创建任务表单字段固定为：
   1. `任务名称`
   2. `执行 agent` 下拉
   3. `优先级` 下拉
   4. `确认任务目标`
   5. `预期产物（选填）`
   6. `上游任务（可选，支持搜索添加）`
   7. `产物交付类型`：`无交付人 / 指定交付人`
   8. `交付给 agent`（仅指定交付人时显示）
6. 当前创建任务表单明确不做：
   1. `下游任务` 直填
   2. `提交说明`
   3. 顶部大段解释文案
7. 任务图必须实现当前核心规则：
   1. 已执行任务位于未执行任务之上
   2. 阻塞任务属于未执行任务
   3. 节点名称固定在图形右侧
   4. 左侧独立时间列展示完成时间
   5. 历史任务支持分批加载
8. 任务详情必须实现：
   1. 基础摘要
   2. 任务管理：`人工修改执行状态 / 重跑任务 / 删除任务`
   3. 回执展示
   4. 产物状态与落盘路径
   5. 动作统一收敛在 `任务管理` 中，不再拆独立动作区块
9. 产物交付第一版必须实现：
   1. 不新增独立左侧 `产物管理` 模块
   2. `设置` 页在既有内容中新增 `产物根路径` 区块
   3. `设置` 页明确展示 `任务记录目录=<根路径>/workspace/`
   4. 无交付人：`<根路径>/<工作agent>/product/<任务名>/...`
   5. 指定交付人：同时写 `product/` 和 `<根路径>/<交付agent>/receive/<任务名>/...`
   6. 所有创建的任务记录统一写入 `<根路径>/workspace/`
   7. 未完成产物交付时不得人工修改为成功
10. 删除管理动作必须实现：
   1. `清空任务` 作用于当前任务图中的全部活动任务与依赖边
   2. `删除任务` 作用于当前选中的单个任务
   3. 两个动作都必须显式确认
   4. 运行中的任务不得删除
   5. 存在运行中任务时不得清空
   6. 删除单个任务后，需按“摘除中间节点”语义自动桥接其原有上游与下游
   7. 多上游/多下游时补齐有效桥接组合，并去重、拒绝形成循环
   8. 删除/清空都必须保留删除留痕，不做静默物理抹除
11. 后端必须补齐当前首期最小闭环：
   1. 任务图创建/读取
   2. 单节点创建
   3. 单节点删除
   4. 整图清空
   5. 依赖边保存与反向下游读取
   6. 调度状态读取
   7. `dispatch-next`
   8. `pause / resume`
   9. `override-status / rerun`
12. 优先级必须保持前后端一致：
   1. 页面展示：`P0/P1/P2/P3`
   2. 后端归一化：`0/1/2/3`

## 4. 明确禁止
1. 不要只做静态前端 mock。
2. 不要把当前创建任务表单改成“上下游都直填”。
3. 不要额外扩出独立产物模块或独立删除管理页。
4. 不要把删除/清空做成无确认的危险操作。
5. 不要在有运行中任务时放行 `清空任务` 或 `删除任务`。
6. 不要脱离现有 `workflow` 视觉语言另起一套风格。
7. 不要遗漏 `workspace/` 任务记录目录规则与删除留痕规则。

## 5. 推荐实现顺序
1. 先接入 `任务中心` 页面和路由。
2. 再落任务图主结构、右侧详情联动和页头动作。
3. 再落创建任务抽屉表单与产物交付字段。
4. 再落 `设置` 中的产物根路径与 `workspace/` 目录配置展示。
5. 再补后端接口与数据落盘：
   1. 图创建/读取
   2. 节点创建/删除
   3. 清空任务图
   4. 调度控制
   5. `override-status / rerun`
   6. 产物交付与 `workspace/` 任务记录
6. 最后联调删除和清空约束：
   1. 运行中不可删
   2. 有运行中任务不可清空
   3. 删除后自动桥接上下游并重算依赖
   4. 多上游/多下游桥接去重且不形成循环
   5. 审计与 `workspace/` 记录留痕
