# 截图索引与归档规范

- 更新时间：2026-03-06
- 目标：把每次模块验证截图放到固定路径，便于验收、复盘与交接时快速检索。

## 1. 统一目录
```text
docs/workflow/screenshots/
  <module>/
    <scene>-overview-current-vN.png
    <scene>-ac-<case>-current-vN.png
```

## 2. 命名规则
1. `module`：功能模块英文短名（示例：`training-center`）。
2. `scene`：场景短名（示例：`agents`、`ops`、`role-portrait`）。
3. `overview-current`：模块当前总览图，只保留当前有效版本。
4. `ac-<case>-current`：验收或专项问题当前有效图，只保留当前有效版本。
5. `vN`：同一截图被重新验证时递增版本号，旧版本不保留在 `docs/`。

## 3. 快速查找
1. 按模块查看：`docs/workflow/screenshots/<module>/`
2. 按场景筛选：同模块内按 `scene` + `current` + `vN` 识别当前有效截图。
3. 全量检索（PowerShell）：
```powershell
Get-ChildItem docs/workflow/screenshots -Recurse -File
```
4. 反查文档引用（PowerShell）：
```powershell
rg -n "docs/workflow/screenshots/" docs/workflow logs/runs
```

## 4. 固定模块目录
1. 对话模块：`docs/workflow/screenshots/chat/`
2. 工作记录与训练：`docs/workflow/screenshots/training-workflow/`
3. 训练中心：`docs/workflow/screenshots/training-center/`
4. 设置模块：`docs/workflow/screenshots/settings/`

## 5. 清理口径
1. `docs/workflow/screenshots/` 只保留当前仍被文档引用的有效截图。
2. 同一场景的历史验证图、改动前后对比图、临时排查图，不保留在 `docs/`。
3. 历史截图、批次证据、原始导出统一保留在 `.output/evidence/`。
4. 文档若仍引用旧图，先更新引用，再删除旧图。

## 6. 新增截图操作（每次验证后）
1. 在 `.output/` 生成截图。
2. 复制到 `docs/workflow/screenshots/<module>/`，使用当前有效命名。
3. 若替换同一场景截图，优先更新引用并清理旧版本，不修改本文档。
4. 若为门禁验收结果，同时在 `logs/runs/` 记录本次 `.output/evidence/` 目录路径。
