# 需求详情-任务中心与依赖可视化编排

## 1. 主题目标与价值说明
1. 在 `../workflow` 左侧导航中新增 `任务中心`，并与 `对话 / 工作记录与训练 / 训练中心 / 设置` 共同形成 5 个顶层模块。
2. 让 workflow 操作者围绕“任务图”进行编排，而不是先进入任务列表再下钻。
3. 让前后依赖关系通过任务节点图直接可读，并支持点击节点展开详情。
4. 让“创建任务”以当前页面内的抽屉表单完成，不切出独立创建页。
5. 为后续其他 workflow 通过接口建图、派发任务和读取图状态预留稳定对象模型。

## 2. 用户画像与使用场景
1. 用户画像 A：workflow 操作者
   1. 关注点：直接创建任务节点、选择上游任务、查看阻塞与推进状态。
2. 用户画像 B：平台治理者
   1. 关注点：依赖门禁是否严格、优先级调度是否一致、留痕是否完整。
3. 用户画像 C：外部 workflow 调用方
   1. 关注点：能否通过接口创建任务图、写入任务节点并读取当前状态。
4. 核心场景：
   1. 用户进入 `任务中心`，直接查看当前任务图。
   2. 用户点击 `+ 创建任务`，在抽屉表单中填写任务名称、执行 agent、优先级、确认任务目标，并可选填写预期产物。
   3. 用户可通过搜索方式为新任务添加上游任务；若没有上游，则该任务进入优先级调度队列。
   4. 当前创建任务表单不直接指定下游任务；下游关系由后续任务引用当前任务为上游后反向展示。
   5. 用户点击任务图中的任意节点，右侧展开该任务的详情、状态、回执展示与任务管理动作。
   6. 外部 workflow 后续通过接口写入任务图后，页面内继续以同一任务图视角展示与推进。

## 3. 用户旅程或关键流程
1. 用户进入 `任务中心`。
2. 页面首先展示页头调度状态与动作区、中部任务图、右侧任务详情，而不是任务列表。
3. 用户点击 `+ 创建任务`，页面以抽屉方式展开创建任务表单。
4. 用户填写基础信息：
   1. 任务名称；
   2. 执行 agent（下拉选择）；
   3. 优先级（下拉选择，页面展示为 `P0/P1/P2/P3`）；
   4. 确认任务目标；
   5. 预期产物（选填）。
5. 用户按需搜索并添加上游任务；当前表单不直接录入下游任务。
6. 任务保存后：
   1. 若存在上游，则生成依赖关系并等待上游完成；
   2. 若不存在上游，则按优先级进入调度队列。
7. 用户在任务图中查看节点关系，理解哪些任务已完成、哪些任务正在执行、哪些任务被阻塞。
8. 用户点击某个任务节点，右侧展开该任务详情。
9. 系统根据依赖门禁、单 agent 单并发和全局并发上限判断哪些任务可进入 `ready/running`。
10. 用户通过 `任务管理` 人工推进任务状态：
   1. `人工修改执行状态` 用于手动回写成功或失败，且操作理由必填；
   2. 当目标状态为成功时，可选填写产出引用；
   3. 当目标状态为成功时，必须先完成产物交付；
   4. 当目标状态为失败时，首期先阻塞其下游任务；
   5. 失败任务在 `任务管理` 中额外提供 `重跑任务`。
11. 当前置任务成功后，系统自动重算后继任务是否从 `pending` 进入 `ready`，并在上游详情中反向展示其下游任务。
12. 用户可在页头执行 `清空任务`，一次性清空当前任务图中的全部活动任务。
13. 用户可在右侧详情中对选中任务执行 `删除任务`。
14. `删除任务` 与 `清空任务` 都必须保留删除留痕，不得静默抹除审计记录。

## 4. 功能需求清单
1. FR-AS-01 顶层模块结构
   1. 左侧导航必须包含且仅包含以下 5 个顶层模块：
      1. `对话`
      2. `工作记录与训练`
      3. `训练中心`
      4. `任务中心`
      5. `设置`
   2. `任务中心` 必须作为独立顶层模块存在。
2. FR-AS-02 主显示结构
   1. 页面必须采用与现有系统一致的单窗口工作台结构。
   2. 主显示区至少包含：
      1. 页头调度状态与动作区；
      2. 中央任务图区；
      3. 右侧任务详情与动作区。
   3. 不再要求任务列表作为主显示区组成部分。
3. FR-AS-03 创建任务入口与容器
   1. 页面顶部必须提供明确的 `+ 创建任务` 入口。
   2. 点击后必须以抽屉或等效侧滑表单展开，不做整页跳转。
   3. 打开创建任务表单后，底层任务图仍保持可见。
4. FR-AS-04 创建任务表单字段
   1. 当前创建任务表单至少包含：
      1. `任务名称`；
      2. `执行 agent`；
      3. `优先级`；
      4. `确认任务目标`；
      5. `预期产物（选填）`；
      6. `上游任务（可选，支持搜索添加)`；
      7. `产物交付类型`（`无交付人 / 指定交付人`）；
      8. `交付给 agent`（仅指定交付人时显示）。
   2. `执行 agent` 必须使用下拉选择，候选范围来自 `训练中心` 当前展示的角色池。
   3. `优先级` 必须使用下拉选择，页面展示口径固定为 `P0/P1/P2/P3`。
   4. 当前表单不得直接提供 `下游任务` 录入字段。
   5. 当前表单不得保留 `提交说明` 或与其等价的额外说明字段。
   6. 产物交付增量规则详见 `docs/workflow/requirements/需求详情-任务中心产物交付与路径管理.md`。
5. FR-AS-05 任务图基础字段
   1. 当前任务图至少包含：
      1. `graph_name`
      2. `source_workflow`
      3. `summary`
      4. `review_mode`
      5. `global_concurrency_limit`
   2. 首期 `review_mode` 固定为 `none`。
6. FR-AS-06 任务节点字段
   1. 每个任务节点至少包含：
      1. `node_name`
      2. `assigned_agent_id`
      3. `node_goal`
      4. `expected_artifact`（选填）
      5. `status`
      6. `priority`
      7. `completed_at`（选填）
   2. 页面显示的 `P0/P1/P2/P3` 必须可稳定映射到节点优先级字段。
   3. `assigned_agent_id` 的候选范围必须来自 `训练中心` 当前展示的角色池。
7. FR-AS-07 上游选择与反向下游展示
   1. 创建任务时必须支持直接指定上游任务。
   2. 任务允许没有上游任务。
   3. 当前创建任务表单不直接指定下游任务。
   4. 当其他任务把当前任务作为上游后，系统必须在任务图和右侧详情中反向展示该任务的下游关系。
   5. 系统必须阻止循环依赖保存。
8. FR-AS-08 无上游任务的优先级调度
   1. 若任务没有上游，则该任务按优先级进入独立调度队列。
   2. 首期页面创建的任务默认优先级应允许用户显式选择。
   3. 推断/假设（置信度: 中）：若多个无上游任务优先级相同，后端应采用稳定次序继续派发，建议按创建顺序处理。
9. FR-AS-09 任务图可视化
   1. 中央区域必须以任务图形式展示节点和依赖边。
   2. 图上的每个节点都只代表一个任务。
   3. 视觉语言应继承现有训练路径/训练演进图的读图方式。
   4. 任务图至少区分：
      1. 已完成任务；
      2. 进行中任务；
      3. 被阻塞任务；
      4. 执行失败任务；
      5. 待开始任务。
   5. 已执行任务必须位于未执行任务之上。
   6. 左侧时间戳需要作为独立时间列展示任务完成时间。
10. FR-AS-10 点击节点展开详情
   1. 点击任务图中的任意任务节点后，右侧详情区必须展开对应任务的信息。
   2. 右侧至少展示：
      1. 任务名称；
      2. 执行 agent；
      3. 当前状态；
      4. 优先级；
      5. 上游任务；
      6. 下游任务；
      7. 执行链路；
      8. 最近回执；
      9. 产物状态与实际落盘路径；
      10. 任务管理动作。
11. FR-AS-11 页头调度控制
   1. 页头必须展示全局调度状态。
   2. 全局状态至少包含：
      1. `运行中`
      2. `暂停中`
      3. `已暂停`
      4. `未启动`
   3. 页头必须展示 `并行agent数`。
   4. 页头必须提供 `暂停调度` 与 `继续运行` 入口。
12. FR-AS-12 依赖门禁
   1. 当任务存在未完成上游时，不得进入 `running`。
   2. 该任务必须显示为 `pending` 或 `blocked`，并明确指出依赖来源。
13. FR-AS-13 单 agent 单并发
   1. 同一个 agent 在任意时刻只允许有 1 个 `running` 任务。
   2. 若该 agent 已有运行中任务，新任务即使依赖满足，也不得直接进入 `running`。
14. FR-AS-14 全局并发上限
   1. 系统必须支持全局并发上限配置。
   2. 当运行中任务数达到上限时，新的就绪任务不得进入 `running`。
15. FR-AS-15 真实执行化升级说明
   1. 从 2026-03-16 起，`任务中心` 不再停留在纯流程流转模式。
   2. 任务详情需新增 `执行链路` 卡片，展示完整提示词、执行过程与结果证据。
   3. 真实执行、设置页任务执行配置与运行批次可见性的详细契约，以 `docs/workflow/requirements/需求详情-任务中心真实执行化与执行链路可视化.md` 为准。
16. FR-AS-16 人工修改执行状态
   1. 任务详情的 `任务管理` 中必须提供 `人工修改执行状态` 入口。
   2. 操作理由必填。
   3. 当目标状态为成功时，`result_ref` 选填，且必须先完成产物交付。
   4. 当目标状态为失败时，其下游任务必须进入阻塞态。
17. FR-AS-17 重跑任务
   1. 失败任务的 `任务管理` 中必须提供 `重跑任务` 入口。
   2. `重跑任务` 仅对失败任务开放。
   3. 重跑后任务重新进入调度或等待依赖校验。
18. FR-AS-18 审计与留痕
   1. 创建任务、编辑上游、编辑优先级、人工改为成功、人工改为失败、重跑、删除、状态推进等关键动作必须留痕。
   2. 留痕至少包含：操作者、时间、对象、动作、理由、引用路径。
19. FR-AS-19 默认阅读顺序
   1. 用户进入页面后，应按以下顺序理解页面：
      1. 页头调度状态与可用动作；
      2. 中央任务图；
      3. 当前选中任务的详情；
      4. 当前可执行动作。
20. FR-AS-20 原型风格继承
   1. 本模块原型与未来页面实现必须继承现有 `workflow` 既有颜色、字号、分栏与滚动逻辑。
   2. 不得切换到陌生的看板式或深色项目管理风格。
21. FR-AS-21 清空任务入口
   1. 页头动作区必须提供 `清空任务` 入口。
   2. `清空任务` 的作用范围是当前任务图中的全部活动任务与依赖边。
   3. 执行前必须经过显式确认。
   4. 当当前任务图仍存在 `running` 任务时，不得允许直接清空。
22. FR-AS-22 单任务删除入口
   1. 右侧任务详情区必须提供 `删除任务` 入口。
   2. 该入口仅作用于当前选中的单个任务。
   3. 执行前必须经过显式确认。
   4. 删除后必须同步摘除与该任务相关的依赖边，并重算受影响任务的上下游关系。
   5. 若被删任务同时存在上游和下游，则必须按“摘除中间节点”的语义自动桥接其原有上游与下游。
   6. 推断/假设（置信度: 中）：若被删任务存在多个上游与多个下游，则应补齐有效桥接组合，并去重、拒绝形成循环。
   7. 运行中的任务不得直接删除。
23. FR-AS-23 删除留痕
   1. `删除任务` 与 `清空任务` 必须保留操作留痕。
   2. 首期默认理解为：从当前任务图活动视图中移除，但审计记录与任务记录删除状态仍可追溯。
24. FR-AS-24 双路径设置与本地任务动态加载
   1. `设置` 页必须同时展示两个工作路径：
      1. `agent路径`：用于扫描并加载该路径下全部 agent；
      2. `任务产物路径`：用于动态加载任务图、任务详情、执行链路与任务产物。
   2. `任务中心` 页面中的任务图、任务详情、完整提示词、执行过程与结果证据，均应通过 `任务产物路径` 动态加载。
   3. `workflow` 仅允许在内存中保留运行中的调度工作集，不得在自身目录中持久化任务明文。

## 5. 非功能需求
1. 可用性：
   1. 用户进入页面后，应在 3 秒内理解“当前调度状态、当前任务图、当前选中任务和下一步动作”。
2. 性能：
   1. 单张任务图在 50 个节点、80 条边基线下应可流畅渲染。
   2. 历史任务需支持按批次增量加载，而不是一次性拉全量。
3. 可靠性：
   1. 依赖门禁必须严格，禁止绕过上游任务直接进入执行态。
   2. 并发门禁必须稳定生效，不得因刷新或重复点击导致同 agent 并发执行。
4. 可观测性：
   1. 任务状态变化必须有事件留痕与审计记录。
5. 一致性：
   1. 布局、配色、信息层级必须与现有 `workflow` 工作台设计语言一致。
   2. 原型、需求文档与后端字段口径必须保持一致。
6. 隐私边界：
   1. `workflow` 页面可显示任务明文，但其展示来源必须是用户本地 `任务产物路径` 的动态加载，而不是 `workflow` 本地持久化副本。

## 6. 验收标准
1. AC-AS-01 顶层模块完整
   1. Given 用户打开 `workflow`
   2. When 左侧导航渲染完成
   3. Then 可看到 `对话 / 工作记录与训练 / 训练中心 / 任务中心 / 设置` 5 个模块
2. AC-AS-02 任务图主视图
   1. Given 用户进入 `任务中心`
   2. When 页面加载完成
   3. Then 可同时看到页头调度状态与动作区、中部任务图、右侧任务详情
   4. And 不再以任务列表作为默认主视图
3. AC-AS-03 创建任务抽屉字段
   1. Given 用户点击 `+ 创建任务`
   2. When 创建任务表单展开
   3. Then 可看到 `任务名称 / 执行 agent / 优先级 / 确认任务目标 / 预期产物（选填） / 上游任务 / 产物交付类型`
   4. And `执行 agent` 与 `优先级` 均为下拉选择
   5. And `上游任务` 支持搜索并添加
   6. And `产物交付类型` 只包含 `无交付人 / 指定交付人` 两种情况
   7. And 选择 `指定交付人` 时出现 `交付给 agent`
   8. And 不直接出现 `下游任务` 或 `提交说明`
4. AC-AS-04 无上游按优先级调度
   1. Given 某任务没有上游
   2. When 任务保存成功
   3. Then 该任务按优先级进入调度队列
   4. And 页面优先级口径展示为 `P0/P1/P2/P3`
5. AC-AS-05 多依赖门禁
   1. Given 某任务存在两个上游
   2. When 仅完成其中一个上游
   3. Then 该任务仍不得进入 `running`
6. AC-AS-06 反向下游展示
   1. Given 任务 B 将任务 A 作为上游保存成功
   2. When 用户选中任务 A
   3. Then 右侧详情中可看到任务 B 被展示为任务 A 的下游
7. AC-AS-07 点击节点展开详情
   1. Given 任务图存在多个任务节点
   2. When 用户点击任一任务节点
   3. Then 右侧详情区联动展示该任务的状态、优先级、上下游、执行链路、回执、产物与任务管理动作
8. AC-AS-08 人工修改成功态受控
   1. Given 某任务处于可人工修改执行状态
   2. When 用户将目标状态改为成功但未填写操作理由
   3. Then 系统拒绝提交并提示理由必填
   4. And `result_ref` 为空时仍允许在理由完整后提交成功
   5. And 若未完成产物交付，则仍不得改为成功
9. AC-AS-09 失败阻塞与任务管理完整
   1. Given 某任务存在下游任务
   2. When 用户通过 `人工修改执行状态` 将该任务改为失败
   3. Then 其下游任务进入阻塞态，并明确显示阻塞来源
   4. And 该失败任务的 `任务管理` 中可看到 `重跑任务` 与 `人工修改执行状态`
10. AC-AS-10 验收证据完备性
   1. Given 任一 AC 进入验收回传
   2. When 提交验收证据
   3. Then 每条 AC 至少包含：页面截图、接口请求/响应示例、关键日志或数据库记录路径、对应代码路径
   4. And `AC-AS-02/03/04/05/06/07/08/09` 需附录屏或等效 GIF 证据
11. AC-AS-11 清空任务受控
   1. Given 当前任务图存在活动任务
   2. When 用户点击 `清空任务`
   3. Then 系统先要求显式确认
   4. And 若存在 `running` 任务，则拒绝清空并给出明确提示
   5. And 若不存在 `running` 任务，则清空当前任务图中的活动任务与依赖边
12. AC-AS-12 单任务删除受控
   1. Given 用户已选中一个非运行中的任务
   2. When 用户点击 `删除任务`
   3. Then 系统先要求显式确认
   4. And 删除后该任务从当前任务图中移除
   5. And 与其相关的依赖边被同步摘除并重算上下游展示
   6. And 若该任务原本位于某条依赖链中间，则其上游与下游被自动桥接
13. AC-AS-13 任务中心从本地任务路径动态加载
   1. Given 用户已在 `设置` 中配置 `agent路径` 与 `任务产物路径`
   2. And `任务产物路径` 下已存在至少一个任务目录
   3. When 用户进入或刷新 `任务中心`
   4. Then 页面可从 `任务产物路径` 动态加载任务图、任务详情与执行链路
   5. And 调度链路仍然清晰可见

## 7. 边界条件与异常处理
1. 若任务没有上游：
   1. 任务直接按优先级进入调度队列。
2. 若多个无上游任务优先级相同：
   1. 推断/假设（置信度: 中）：后端应采用稳定次序继续派发，建议按创建顺序处理。
3. 若出现循环依赖：
   1. 系统必须拒绝保存，并指出涉及的节点链路。
4. 若用户选中的执行 agent 已不在训练中心角色池中：
   1. 页面必须显示异常状态并阻止继续派发。
5. 若页面优先级标签与后端优先级值不一致：
   1. 系统必须拒绝保存或返回明确映射错误，不得静默写入错误优先级。
6. 若任务失败：
   1. 首期只允许阻塞，不允许自动恢复。
7. 若全局并发上限被调低到小于当前运行数：
   1. 不中断已有运行任务，但新任务不得继续进入 `running`。
8. 若用户尝试删除运行中的任务：
   1. 系统必须拒绝，并提示需等待任务结束或先完成状态回写。
9. 若用户尝试清空包含运行中任务的任务图：
   1. 系统必须拒绝，并提示存在运行中任务时不可清空。
10. 若被删除任务同时存在多个上游与多个下游：
   1. 推断/假设（置信度: 中）：系统应在删除后补齐有效桥接边，并拒绝生成重复边或循环边。
11. 若 `任务产物路径` 不可访问或目录结构损坏：
   1. 页面必须明确提示加载失败原因，不得静默回退为读取 `workflow` 本地持久化副本。

## 8. 依赖项与开放问题
1. 依赖项：
   1. 依赖 `训练中心` 当前角色池可被稳定读取，作为任务节点的执行 agent 候选来源。
   2. 依赖 `workflow` 后端具备独立的任务图读写、调度与审计留痕能力。
   3. 依赖现有训练路径风格可作为视觉继承基线。
2. 开放问题：
   1. 首期是否允许任务在进入 `running` 前再次改派执行 agent。
      1. 推断/假设（置信度: 中）：建议允许，但仅限未进入 `running` 的任务。
   2. 首期是否允许任务在创建后再次编辑上游关系。
      1. 推断/假设（置信度: 中）：建议允许，但需重新校验循环依赖与阻塞关系。
