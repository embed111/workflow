# 截图索引与归档规范

- 更新时间：2026-03-05
- 目标：把每次模块验证截图放到固定路径，便于验收、复盘与交接时快速检索。

## 1. 统一目录
```text
docs/workflow/screenshots/
  <module>/
    <scene>-<before|after>-vN.png
```

## 2. 命名规则
1. `module`：功能模块英文短名（示例：`training-center`）。
2. `scene`：场景短名（示例：`agents`、`ops`）。
3. `before|after`：改动前后。
4. `vN`：同场景多次修订用版本号（`v1`、`v2`）。

## 3. 快速查找
1. 按模块查看：`docs/workflow/screenshots/<module>/`
2. 按场景筛选：同模块内按 `scene` + `before/after` + `vN` 识别。
3. 全量检索（PowerShell）：
```powershell
Get-ChildItem docs/workflow/screenshots -Recurse -File
```

## 5. 新增截图操作（每次验证后）
1. 在 `.output/` 生成截图。
2. 复制到 `docs/workflow/screenshots/<module>/` 并按命名规则重命名。
3. 在本文档追加对应模块索引行（可在标题或备注里保留验证日期）。
4. 若为门禁验收结果，同时在 `logs/runs/` 记录本次截图目录路径。
