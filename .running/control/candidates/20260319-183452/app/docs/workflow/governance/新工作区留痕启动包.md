# 新工作区留痕启动包（失败后可回放）

## 1. 目标
1. 降低首次实践失败成本。
2. 保证“失败可复盘、结论可追溯、二次迭代可重启”。

## 2. 新工作区最小目录
```text
workflow/
  docs/
    workflow/
      overview/
        需求概述.md
      requirements/
        需求详情-训练闭环监控系统-Phase0.md
      design/
      governance/
      reports/
      prompts/
  logs/
    events/
      events-YYYYMMDD.jsonl
    decisions/
      decisions-YYYYMMDD.md
    runs/
      run-YYYYMMDD-HHMM.md
    summaries/
      daily-summary.md
      failure-cases.md
  state/
    session-snapshot.md
    change-log.md
  incidents/
    INC-YYYYMMDD-001.md
  metrics/
    cli-baseline-latency.json
    workflow-latency-daily.json
```

## 3. 每日必须留痕（最小 5 项）
1. 新增会话总数。
2. 待分析与待训练任务数。
3. 当日切换记录（是否发生 A/B 切换、耗时）。
4. 当日失败 Top3 原因标签。
5. 次日第一优先修复项。

## 4. 事件日志最小契约（JSONL）
```json
{
  "event_id": "evt-20260224-001",
  "timestamp": "2026-02-24T10:00:00+08:00",
  "session_id": "sess-001",
  "actor": "user|workflow|agent|trainer",
  "stage": "chat|analyze|train|switch|deploy",
  "action": "send_message|decision|run_training|ab_switch",
  "status": "success|failed",
  "latency_ms": 1234,
  "task_id": "REQ-20260224-001",
  "reason_tags": ["missing_acceptance"],
  "ref": "logs/runs/run-20260224-1000.md"
}
```

```md

- current_phase: phase0
- current_focus: <一句话>

3. 最大阻塞项

## Key Metrics

4. ab_switch_count:
5. critical_failures:

## Next Start
1. 下次启动先看哪个日志

```

## 6. `incidents/INC-*.md` 模板
```md
# Incident - INC-YYYYMMDD-001

## 1. 现象
- 发生了什么

## 2. 影响
- 影响范围
- 持续时间

## 3. 时间线
1. T0
2. T1
3. T2

## 4. 根因（事实/推断分开）
1. 事实:
2. 推断/假设（置信度: 高/中/低）:

## 5. 临时止血
1. ...

## 6. 长期修复
1. ...

## 7. 验收
1. 如何证明修复有效
```

## 7. 每日收工动作（不超过 5 分钟）
1. 更新 `logs/summaries/daily-summary.md`。
2. 更新 `state/session-snapshot.md`。
3. 若有失败，新增一条 `incidents/INC-*.md`。
4. 把当天关键事件汇总到 `state/change-log.md`。

## 8. 失败后带回分析的最小打包清单
1. `docs/workflow/overview/需求概述.md`
2. `docs/workflow/requirements/需求详情-训练闭环监控系统-Phase0.md`
3. `logs/summaries/daily-summary.md`
4. `logs/summaries/failure-cases.md`
5. 最近 1 个 `INC-*.md`
6. 最近 1 天 `events-YYYYMMDD.jsonl`
7. `state/session-snapshot.md`

## 9. 模块验证截图入口
1. 截图索引文档：`docs/workflow/governance/截图索引与归档规范.md`。
2. 截图根目录：`docs/workflow/screenshots/`（按模块保留当前有效截图，历史批次保留在 `.output/evidence/`）。
3. 每次验证结束后，把 `.output/` 新截图复制到上述归档目录并更新索引文档。
