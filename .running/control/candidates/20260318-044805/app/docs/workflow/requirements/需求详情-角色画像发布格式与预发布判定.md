# 需求详情-角色画像发布格式与预发布判定

## 1. 主题目标与价值说明
1. 主题目标：在训练中心选择工牌后，优先展示“简历式角色画像”（一眼看懂该 agent 能做什么），并将“发布版本/普通提交/预发布”语义收敛为可判定规则。
2. 业务价值：
   1. 降低版本可用性判断歧义，避免把不合规提交误当作“发布版本”。
   2. 让角色信息展示更贴近使用决策（先看能力与知识，再做发布/切换动作）。
   3. 头像能力收敛为“单头像展示 + 更换入口”，降低界面噪音并保留基本运营能力。
   4. 让角色详情优先展示“最新正式发布版本的第一人称角色述职报告”，增强对外可理解性。

## 2. 用户画像与使用场景
1. 用户画像 A：训练中心操作者
   1. 关注点：选中角色后第一时间知道“这个角色擅长什么、知道什么、适合什么场景”。
2. 用户画像 B：发布治理人员
   1. 关注点：发布版本必须有统一格式，不合规提交不能进入发布列表。
3. 用户画像 C：产品维护者
   1. 关注点：本地一旦有未忽略改动，系统要明确提示“预发布态”。
4. 用户画像 D：角色运营人员
   1. 关注点：界面只展示一个头像，并保留一个轻量的“更换头像”入口。
5. 用户画像 E：角色详情查看者
   1. 关注点：希望通过角色详情直接理解该 agent 最新正式发布版本的完整能力，而不只看到版本亮点。

## 3. 用户旅程或关键流程
1. 用户进入训练中心“角色资产与版本”。
2. 选择任一工牌后，系统优先判断该 agent 最新正式发布版本是否已绑定发布报告：
   1. 若已绑定，则右侧首屏优先展示该报告中的第一人称角色述职摘要与全量能力清单。
   2. 若未绑定，则回退为结构化角色画像渲染。
3. 右侧按“单栏自上而下”展示简历式角色画像（首屏）：`我是/我当前能做什么`、`全量能力清单`、`知识范围`、`Agent Skills`、`适用场景`。
4. 系统读取候选版本元数据并执行发布格式校验：
   1. 合规 -> 进入发布版本列表。
   2. 不合规 -> 仅作为普通提交，不进入发布版本列表。
5. 系统持续检测工作区状态：
   1. `git status --porcelain --untracked-files=normal` 非空 -> 标记预发布。
6. 用户在角色信息区看到单个头像与“更换头像”入口；点击入口后可选择本地文件，成功后替换当前头像，失败时保留旧头像或默认头像并提示原因。

## 4. 功能需求清单（编号）
1. FR-RP-01 角色画像简历式首屏展示
   1. 在训练中心选中工牌后，角色画像信息必须优先展示在详情区顶部。
   2. 首屏必须包含：`我是/我当前能做什么（第一人称述职摘要）`、`全量能力清单`、`角色知识范围`、`Agent Skills`、`适用场景`、`版本说明`。
   3. 首屏结构应支持“5 秒可读”：无需滚动即可完成“该 agent 能做什么”的主判断。
   4. 若最新正式发布版本已绑定发布报告，首屏应优先读取该报告中的公开介绍字段；无绑定报告时才回退结构化字段。
2. FR-RP-02 发布格式字段门禁
   1. 发布版本必须包含 FR-RP-01 定义的 5 个字段。
   2. 任一字段缺失或空值时，不得认定为发布版本。
3. FR-RP-03 不合规版本降级
   1. 不符合发布格式的记录统一视为“普通提交”。
   2. 普通提交不进入“已发布版本”列表，不参与版本切换候选。
4. FR-RP-04 预发布状态判定
   1. 工作区执行 `git status --porcelain --untracked-files=normal` 返回非空时，角色状态标记为“预发布”。
   2. 判定不区分改动是否已提交。
5. FR-RP-05 .gitignore 边界
   1. 预发布判定仅基于未被 `.gitignore` 忽略的改动。
6. FR-RP-06 单头像展示与更换入口
   1. 在角色信息区仅展示一个头像，不展示独立的头像管理面板、拖拽上传区或多头像候选区。
   2. 头像附近仅提供一个轻量的“更换头像”入口，用于触发本地文件选择。
   3. 更换成功后立即展示新头像；更换失败时保留旧头像或默认头像。
   4. 未设置头像时必须展示默认头像。
   5. 头像更换不作为发布门禁条件。
7. FR-RP-07 角色画像可读性与来源约束
   1. `我能做什么` 必须以短句列表展示，建议 3~5 条，避免长段落。
   2. `Agent Skills` 与 `适用场景` 必须可快速扫读（列表或标签化），不接受纯大段文本。
   3. `skills` 字段的展示语义必须是 `agent skill` 清单，不得改写为抽象化技能标签。
   4. 当前阶段 `Agent Skills` 默认直接映射当前 agent 工作区 `.codex/skills/` 目录下真实存在的技能项。
   5. 当前阶段默认不混入系统级全局 skills，避免把通用能力误展示为该 agent 的专属技能。
   6. 展示形态固定为“单栏阅读流（A）”，不采用双栏摘要卡布局。
   7. 若最新正式发布版本已绑定发布报告，角色详情页必须优先渲染其中的 `first_person_summary/full_capability_inventory/knowledge_scope/agent_skills/applicable_scenarios`。
   8. 角色详情页不得只展示 `capability_delta` 或发布亮点；必须能让未参与开发的查看者理解该 agent 当前正式发布版本完整能做什么。
   9. 在正式发布确认成功前，预发布报告不得覆盖当前正式发布角色详情。
   10. 历史正式发布版本若缺少绑定发布报告，允许回退到结构化字段渲染，并标记当前为回退来源。
8. FR-RP-08 证据口径
   1. 本主题验收需包含功能截图、判定日志/命令输出、代码路径。
   2. 统计截图仅辅助，不替代功能截图。

## 5. 非功能需求（性能、安全、可用性、合规等）
1. 性能：角色切换后画像信息加载 `p95 <= 1.5s`（同环境基线）。
2. 可用性：Git 不可用时页面应降级提示“Git 不可用”，但不阻塞角色信息查看。
3. 一致性：同一角色在同一时刻“发布/普通/预发布”状态判定唯一且可复现。
4. 审计：预发布判定命令输出与时间戳需可追溯。
5. 交互：头像更换失败必须给出可读错误原因（格式不支持、文件过大、网络失败等）。
6. 头像区整洁性：头像相关操作不得形成独立大模块，不得挤占“我能做什么”首屏阅读空间。
7. 头像上传限制（已确认）：仅支持 `png/jpg/webp`，单文件不超过 `2MB`。
8. 角色详情来源透明性：当页面使用“发布报告”或“结构化字段回退”渲染时，必须能区分当前来源。

## 6. 验收标准（Given/When/Then）
1. AC-RP-01 画像优先展示
   1. Given 用户进入训练中心并选择工牌
   2. When 详情区完成渲染
   3. Then 首屏优先展示简历式角色画像信息
   4. And 包含 `我是/我当前能做什么/全量能力清单/角色知识范围/Agent Skills/适用场景/版本说明`
2. AC-RP-02 发布格式通过
   1. Given 某版本包含 5 个发布字段且值非空
   2. When 系统执行发布格式校验
   3. Then 该版本进入发布版本列表
3. AC-RP-03 发布格式不通过
   1. Given 某版本缺失任一发布字段
   2. When 系统执行发布格式校验
   3. Then 该版本不进入发布版本列表
   4. And 该版本仅按普通提交处理
4. AC-RP-04 普通提交不可切换
   1. Given 某记录被判定为普通提交
   2. When 用户查看可切换版本列表
   3. Then 该记录不可被选为切换目标
5. AC-RP-05 预发布判定生效
   1. Given 工作区 `git status --porcelain --untracked-files=normal` 返回非空
   2. When 系统刷新角色状态
   3. Then 角色标记为预发布
6. AC-RP-06 .gitignore 不计入
   1. Given 仅存在被 `.gitignore` 忽略的文件变化
   2. When 系统执行预发布判定
   3. Then 不标记为预发布
7. AC-RP-07 单头像与更换入口
   1. Given 用户进入角色详情页
   2. When 身份头卡完成渲染
   3. Then 页面仅展示一个头像
   4. And 头像附近仅提供一个“更换头像”入口
8. AC-RP-08 更换头像生效
   1. Given 角色未配置头像或已有旧头像
   2. When 用户通过“更换头像”入口选择本地头像文件
   3. Then 成功时立即显示新头像
   4. And 失败时保留旧头像或默认头像并显示失败原因
9. AC-RP-09 Agent Skills 语义正确
   1. Given 角色声明了多个 agent skill
   2. When 页面渲染技能区
   3. Then 展示内容应为 agent skill 清单
   4. And 不得改写为泛化技能标签
10. AC-RP-10 Agent Skills 来源正确
   1. Given 当前 agent 工作区 `.codex/skills/` 下存在多个本地技能
   2. When 页面渲染 `Agent Skills`
   3. Then 展示项应与该目录下真实技能项一致
   4. And 默认不展示非本工作区的系统级通用 skills
11. AC-RP-11 5 秒可读性
   1. Given 用户首次查看角色详情
   2. When 首屏渲染完成
   3. Then 用户无需滚动即可看到“我能做什么”的核心信息块
   4. And 页面以单栏阅读流呈现，不出现双栏并列主体信息区
12. AC-RP-12 最新正式发布报告优先展示
   1. Given 某 agent 最新正式发布版本已绑定发布报告
   2. When 用户查看角色详情
   3. Then 页面优先展示该报告中的第一人称摘要与全量能力清单
   4. And 不回退到旧版结构化字段作为主内容
13. AC-RP-13 预发布不覆盖正式角色详情
   1. Given 某 agent 已生成预发布报告但尚未确认发布
   2. When 用户查看角色详情
   3. Then 页面仍展示上一正式发布版本的角色介绍
   4. And 预发布报告只在发布评审页可见
14. AC-RP-14 历史版本回退渲染
   1. Given 某正式发布版本缺少绑定发布报告
   2. When 用户查看角色详情
   3. Then 页面回退到结构化字段渲染
   4. And 标记当前为“结构化字段回退”来源
15. AC-RP-15 验收证据完备性
   1. Given 本主题进入验收
   2. When 提交验收材料
   3. Then 至少包含功能截图、`git status` 判定输出、代码路径

## 7. 边界条件与异常处理
1. Git 仓库不可用：
   1. 发布/预发布状态显示“不可判定（Git 不可用）”，并提供可读原因。
2. 发布字段解析失败：
   1. 默认降级为普通提交，不得误判为发布版本。
3. 角色画像字段为空：
   1. 前端显示占位文案，并在校验层标记不合规。
4. 未声明 Agent Skills：
   1. 技能区显示“未声明 Agent Skills”，但不得伪造或自动脑补技能内容。
5. 头像资源不可访问：
   1. 自动回退默认头像，不阻断页面功能。
6. 上传文件不合法：
   1. 拒绝写入并返回错误原因，不覆盖现有头像。
7. 最新正式发布报告解析失败：
   1. 自动回退结构化字段渲染，并记录来源降级原因。

## 8. 依赖项与开放问题
1. 依赖项：
   1. 训练中心角色详情区支持扩展字段渲染。
   2. 当前发布版本元数据来源可被读取与解析。
   3. 运行环境可执行 Git 状态命令并稳定返回。
   4. 后端具备头像文件存储与访问能力。
   5. 运行环境可读取当前 agent 工作区 `.codex/skills/` 目录及其 `SKILL.md` 元数据。
   6. 角色详情 API 可读取最新正式发布版本绑定的公开发布报告或结构化快照。
2. 开放问题（待后续讨论）：
   1. 头像存储策略（本地路径/对象存储）与清理策略待与实现侧确认。
   2. 后续是否需要把系统级全局 skills 与本地 `.codex/skills/` 合并展示，待后续阶段确认。
   3. 历史正式发布版本的第一人称角色述职报告是否需要批量补录，待后续阶段确认。
3. 能力扩展评估（skill/agent）：
   1. 新增 agent 角色：`暂不采用`（头像绘制 agent 为后续独立主题）。
   2. 新增 skill：`暂不采用`（当前可由既有需求收敛门禁与文档治理技能覆盖）。
