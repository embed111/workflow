# 需求详情-测试数据展示全局开关统一

## 1. 主题目标与价值说明
1. 主题目标：
   1. 将“测试数据展示”收敛为唯一全局开关，消除会话入口与设置页双开关导致的语义分裂。
   2. 将该开关升级为后端持久化配置，确保跨会话、跨浏览器一致。
2. 业务价值：
   1. 降低用户认知成本，避免“同名能力在不同入口表现不同”。
   2. 降低执行与验收歧义，建立单一事实源（Single Source of Truth）。
   3. 为后续自动化分析与训练计划提供稳定配置基线。

## 2. 用户画像与使用场景
1. 用户画像 A：workflow 日常使用者
   1. 关注点：任何页面都按同一开关语义展示或隐藏测试数据。
2. 用户画像 B：训练运营者
   1. 关注点：训练中心、会话列表、Dashboard 的数据口径一致。
3. 用户画像 C：系统维护者
   1. 关注点：配置可持久化、可回放，避免浏览器本地状态漂移。
4. 核心场景：
   1. 在“全局设置”打开/关闭“展示测试数据”后，回到会话入口，agent 列表立即按同一结果刷新。
   2. 刷新页面或更换浏览器后，系统仍使用同一开关值展示数据。
   3. 训练中心与 Dashboard 查询统一受该开关控制。

## 3. 用户旅程或关键流程
1. 用户进入系统，前端启动时先拉取后端全局配置。
2. 系统将“展示测试数据”值加载到单一运行态字段（仅此一处）。
3. 用户在“全局设置”切换开关，前端调用后端配置写入接口。
4. 写入成功后，系统触发会话入口、Dashboard、会话列表、训练中心联动刷新。
5. 用户刷新页面或切换浏览器再次访问，仍读取到同一配置值。

## 4. 功能需求清单（编号）
1. FR-TD-01 单一全局开关
   1. 系统仅保留一个“展示测试数据”控制项。
   2. 会话入口不得再存在独立的“测试/系统”可见性开关。
2. FR-TD-02 后端持久化配置
   1. 开关值必须由后端保存并提供读取接口。
   2. 配置作用域为实例级全局配置（非会话级、非浏览器级）。
3. FR-TD-03 全模块统一生效
   1. 会话入口 agent 列表（含测试/系统 agent）必须受该开关控制。
   2. Dashboard、会话列表、训练中心相关查询必须受该开关控制。
4. FR-TD-04 启动加载与写回
   1. 前端启动时必须先读取后端配置再渲染对应开关状态。
   2. 开关变更后必须先写后端成功，再刷新各模块数据。
5. FR-TD-05 本地缓存降级策略
   1. 浏览器本地缓存仅可作为短暂回显，不得作为全局真值来源。
   2. 本地缓存与后端值冲突时，以后端值覆盖。
6. FR-TD-06 查询参数统一口径
   1. 页面侧不得再为不同模块拼装不同测试数据参数语义。
   2. 页面 API 调用统一依赖后端全局配置决定是否纳入测试数据。
7. FR-TD-07 向后兼容迁移
   1. 旧的前端本地键（如历史开关键）应在迁移期自动清理或忽略。
   2. 旧入口控件移除后，不得影响核心会话创建流程。
8. FR-TD-08 证据语义门禁（沿用）
   1. 验收必须提交功能截图，不接受仅门禁统计截图。

## 5. 非功能需求（性能、安全、可用性、合规等）
1. 性能：
   1. 开关切换后 2 秒内完成主要页面数据联动刷新。
2. 可靠性：
   1. 后端配置写入失败时不得静默成功，前端需回滚开关状态并提示失败。
3. 可用性：
   1. 启动拉取配置失败时，页面需明确提示“配置读取失败”，并允许重试。
4. 可观测性：
   1. 配置读取/写入动作至少保留最小日志（时间、操作、结果）。

## 6. 验收标准（Given/When/Then）
1. AC-TD-01 唯一入口
   1. Given 用户打开会话入口与全局设置
   2. When 检查“测试数据展示”控制项
   3. Then 仅全局设置存在该开关，会话入口无独立同类开关
2. AC-TD-02 后端持久化
   1. Given 用户在全局设置关闭“展示测试数据”
   2. When 刷新页面或更换浏览器后重新进入系统
   3. Then 开关状态保持关闭，且数据展示口径一致
3. AC-TD-03 会话入口联动
   1. Given 开关从开启切换为关闭
   2. When 返回会话入口 agent 列表
   3. Then 测试/系统 agent 按新状态隐藏
4. AC-TD-04 Dashboard 联动
   1. Given 开关状态变更
   2. When 查看 Dashboard 统计
   3. Then 统计结果按同一开关口径刷新
5. AC-TD-05 训练中心联动
   1. Given 开关状态变更
   2. When 查看训练中心列表与详情
   3. Then 数据是否含测试项与全局开关一致
6. AC-TD-06 写失败回滚
   1. Given 后端配置写入失败
   2. When 用户切换开关
   3. Then 前端回滚到原值并提示失败，不得出现“UI 已变但后端未生效”
7. AC-TD-07 启动失败可见
   1. Given 系统启动时配置读取失败
   2. When 用户进入页面
   3. Then 页面显示明确错误与重试入口
8. AC-TD-08 旧键迁移不残留
   1. Given 浏览器中存在历史本地开关键
   2. When 完成本次版本升级
   3. Then 旧键不再驱动真实行为，系统以全局后端值为准
9. AC-TD-09 验收证据完备性
   1. Given 任一 AC 回传
   2. When 审核证据
   3. Then 至少包含功能截图、API 请求/响应、日志/DB 路径、代码路径；门禁统计截图仅作辅助

## 7. 边界条件与异常处理
1. 后端配置表缺失：
   1. 启动时返回可读错误，不允许前端默默退化为本地真值。
2. 部分模块刷新失败：
   1. 必须提示“部分页面未同步”，并提供手动重试。
3. 配置并发写入：
   1. 以最后一次成功写入为准，并记录时间戳用于排障。

## 8. 依赖项与开放问题
1. 依赖项：
   1. 现有全局设置模块支持新增/扩展后端配置读写接口。
   2. 会话入口、Dashboard、训练中心均可接收统一配置源。
2. 开放问题：
   1. 首次部署默认值是否固定为 `true`（建议沿用当前行为，避免突变）。
3. 能力扩展评估（skill/agent）：
   1. 新增 agent 角色：`暂不采用`（本需求属于单模块配置治理，不涉及角色分工重构）。
   2. 新增 skill：`暂不采用`（当前变更复杂度低，现有文档与门禁模板可覆盖）。
