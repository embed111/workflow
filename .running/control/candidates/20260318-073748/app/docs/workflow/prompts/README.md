# 提示词目录说明

## 目的
1. 将执行提示词集中在 `docs/workflow/prompts/`，避免散落在 `docs/workflow/` 根目录。
2. 保持分发路径稳定，减少跨工作区执行偏差。

## 命名规则
1. 文件名保持：`执行提示词-<主题>-<YYYYMMDD>.md`。
2. 每次调整提示词使用“新增文件”，不覆盖历史文件。

## 保留策略
1. 每次新增后执行一次清理。
2. 仅保留最近 7 轮提示词（按 `LastWriteTime` 倒序）。
3. 超出 7 轮的旧文件直接删除。

## 分发顺序
1. 先同步文件到执行工作区并校验一致性。
2. 再给出可转发口令：
   - `请按照文件《docs/workflow/prompts/<文件名>》中的提示词执行。`

## 证据语义门禁（默认启用）
1. 门禁统计表/汇总矩阵截图只能作为辅助证据，不能替代 AC 对应功能页面截图。
2. 每条 AC 回传至少包含：功能截图、API 请求/响应、日志/DB 路径、代码路径。
