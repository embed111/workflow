# 执行提示词-训练闭环信息架构收口与滚动修正-20260313

请按照以下文件中的最新口径执行本次训练闭环界面收口。本次不是单纯修复滚动条，而是要先落实已经确认的信息架构调整，再完成滚动分层与可见性修正。

## 2. 本次实现目标
1. 左侧任务列表收口为唯一主入口：
   1. `+ 新建训练任务` 进入创建态；
   2. 选择已有任务进入对应任务状态页；
   3. 页面顶部不再保留与其重复的 `创建任务 / 任务状态` 模式切换。
2. 创建页结构收口为“两列”：
   1. 左侧沿用与状态页一致的任务列表骨架；
   2. 中部承载创建表单、页签、训练路径预览与提交区；
   3. 删除右侧 `创建结果与校验` 重复栏目。
3. 创建页动作语义收口：
   1. 左侧 `+ 新建训练任务` 仅表示进入创建态；
   2. 真正提交动作只保留 `保存并启动首轮 / 保存草稿`。
4. 创建页与状态页左栏必须共享同一套骨架：
   1. 搜索框；
   2. 筛选按钮；
   3. 状态筛选行；
   4. `+ 新建训练任务`；
   5. 任务卡、状态标签、指标卡、更新时间；
   6. 差异只允许保留“当前态高亮”。
5. 中栏内部页签必须是真实可切换视图，不允许只做静态示意：
   1. 创建页：`基础信息 / 首轮工作集 / 启动确认`
   2. 状态页：`当前概览 / 工作集变化 / 三轮评测 / 历史记录`
6. 最宽中栏改为统一主滚动容器：
   1. 创建页中栏整体滚动；
   2. 状态页中栏整体滚动；
   3. 不允许出现“已有滚动条但内容仍被压缩/裁切”的视觉 bug。

## 3. 实现理解门禁
1. 不要把本次任务理解为“只调滚动条样式”。
2. 本次首先是信息架构收口，其次才是滚动与可见性修正。
3. 若实现结果仍保留“顶部模式切换 + 左侧任务列表 + 右侧重复校验栏”三套并行控制，则视为未按需求执行。

## 4. 明确禁止
1. 不要恢复顶部 `创建任务 / 任务状态` 模式切换。
2. 不要恢复创建页右侧 `创建结果与校验` 栏。
3. 不要让创建页和状态页左侧任务列表继续演化成两套不同骨架。
4. 不要把“有滚动条”误当作“内容完整可见”；必须处理内容被 `flex-shrink` 压缩或被内层容器裁切的问题。
5. 不要用整页长滚动替代区内滚动。

## 5. 最低验收要求
1. 结构验收：
   1. 左侧任务列表成为唯一主入口；
   2. 创建页为“两列”，状态页保留“左列表 + 中详情 + 右辅助”的阅读结构；
   3. 创建页不再存在右侧 `创建结果与校验` 重复栏；
   4. 页面顶部不再出现 `创建任务 / 任务状态` 重复切换。
2. 视觉验收：
   1. 创建页与状态页左侧任务列表骨架一致；
   2. 中栏底部内容可通过滚动完整查看，不出现半截卡片、底部被截断或内容被压扁；
   3. 页签栏保持可发现性，不做高强度按钮化。
3. 交互验收：
   1. 中部页签点击后真实切换内容；
   2. 切换页签后中栏滚动位置回到顶部；
   3. 不出现整页长滚动替代区内滚动。
4. 证据回传：
   1. 创建页截图：左栏统一骨架 + 两列结构 + 中栏提交区 + 中栏滚动到底部；
   2. 状态页截图：左栏统一骨架 + 中栏滚动到底部；
   3. 页签切换截图：至少 1 张创建页、1 张状态页；
   4. 若改动了前端脚本，请附对应代码路径。
