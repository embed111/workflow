# 执行提示词-定时任务模块与日历视图开工-20260323

请按照文件《docs/workflow/prompts/执行提示词-定时任务模块与日历视图开工-20260323.md》中的提示词执行。

## 1. 先读文件
1. `docs/workflow/overview/需求概述.md`
2. `docs/workflow/requirements/需求详情-定时任务模块与日历视图.md`
3. `docs/workflow/requirements/需求详情-任务中心与依赖可视化编排.md`
4. `docs/workflow/requirements/需求详情-任务中心后端接口与调度规则.md`

6. `docs/workflow/prototypes/定时任务模块与日历视图/定时任务参考图.html`

## 2. 本轮目标
1. 在 `../workflow` 中正式落地一个与 `任务中心` 同级的 `定时任务` 模块。
2. 首期做成“可真实配置、可真实命中、可真实建单、可真实回看结果”的周期任务工作台，不做只会展示的静态壳。
3. 计划负责配置与触发，执行真相源继续归 `任务中心`，不要在 `定时任务` 模块里复制第二套执行系统。

## 3. 你必须按以下口径实现
1. 左侧导航当前有效口径固定为：
   1. `对话`
   2. `工作记录与训练`
   3. `训练中心`
   4. `任务中心`
   5. `定时任务`
   6. `需求与缺陷`
   7. `设置`
2. `定时任务` 必须是独立顶层模块，不挂在 `任务中心` 二级页签下。
3. 模块内容区必须只保留两个顶层视角：
   1. `列表视角`
   2. `日历视角`
4. `列表视角` 默认采用“左侧计划列表 + 右侧详情”的双栏结构。
5. `新建定时任务` 入口固定放在左侧 `定时任务列表` 面板右上角，使用轻量 `+` 图标按钮，不放在模块页头主标题区。
6. 右侧详情必须直接可见以下信息：
   1. 基础配置
   2. 发起任务内容预览
   3. 触发规则
   4. 未来触发
   5. 最近执行结果
   6. 关联任务中心实例
   7. `编辑计划 / 启用或停用 / 删除计划`
7. `日历视角` 必须同时展示：
   1. 未来计划触发
   2. 已触发实例的实际执行结果
8. 日历右侧详情区必须可直接：
   1. 查看本次将发起的任务摘要或关键动作
   2. 点击 `编辑计划`
9. 日历首期固定为 `月视图 + 选中日详情`，但必须支持：
   1. `上一月`
   2. `下一月`
   3. 切月后同步刷新月份标题、日期格事件和右侧详情上下文
10. 首期所有规则命中与日历展示默认统一使用 `北京时间（Asia/Shanghai）`。
11. 单条计划必须支持混合启用以下四类规则：
   1. `每月`
   2. `每周`
   3. `每日`
   4. `定时`
12. 同一计划若有多条规则在同一分钟同时命中，必须只创建 1 条新的任务中心单任务实例。
13. 去重后仍必须保留触发审计信息，至少可追溯：
   1. `source_schedule_id`
   2. `planned_trigger_at`
   3. `trigger_instance_id`
   4. `trigger_rule_summary`
14. 每次计划命中后，系统都必须：
   1. 向 `任务中心` 创建新的单任务实例
   2. 写入计划来源字段
   3. 带上 `launch_summary / execution_checklist / done_definition` 快照
   4. 立即请求进入既有调度/运行流程
15. 若目标 agent 忙碌、存在依赖门禁或命中全局并发上限：
   1. 实例仍然创建成功
   2. 真实等待/调度状态继续由 `任务中心` 判定
   3. `定时任务` 只同步展示结果，不得伪装为已运行
16. 定时计划必须支持完整 CRUD 与启停：
   1. 新增
   2. 编辑
   3. 删除
   4. 启用
   5. 停用
17. 删除计划只影响未来触发，不得抹除历史触发记录与已创建实例的追溯链路。

## 4. 明确禁止
1. 不要把 `定时任务` 退化成只保存 cron 文本的配置页。
2. 不要把一次计划命中直接展开成整张任务图模板。
3. 不要绕过 `任务中心` 的依赖门禁、单 agent 单并发和全局并发上限。
4. 不要只展示未来计划，不展示真实执行结果。
5. 不要把默认时间基准做成多时区共享方案。
6. 不要在首期混入节假日跳过、停机补跑、复杂 cron 表达式。

## 5. 推荐实现顺序
1. 先补左侧导航、模块路由、列表/日历双视角骨架。
2. 再补定时计划数据模型、CRUD、启停与规则编辑。
3. 再补命中扫描、同分钟去重、建单与请求调度链路。
4. 最后补日历结果叠加、审计留痕、联动跳转与自动化测试。

## 6. 最低验收回传
1. 页面截图：
   1. 列表默认态
   2. 列表详情态
   3. 创建或编辑态
   4. 日历月视图
   5. 日历切月后视图
   6. 命中后显示真实等待或执行结果的详情态
2. 接口与行为证据：
   1. 定时计划新增请求/响应
   2. 定时计划编辑请求/响应
   3. 启用/停用请求/响应
   4. 同计划同分钟去重命中证据
   5. 创建任务中心实例请求/响应
   6. 请求调度或运行的请求/响应
   7. 任务中心返回等待或运行状态的读取证据
3. 日志或数据证据：
   1. 计划命中日志
   2. 去重日志
   3. 建单日志
   4. 调度请求日志
   5. 来源字段持久化位置或数据样例
4. 代码落点：
   1. `定时任务` 页面入口
   2. 计划 CRUD 实现位置
   3. 命中扫描与去重实现位置
   4. 与 `任务中心` 的建单和调度联动实现位置
5. 风险与未完成项：
   1. 若首版只完成部分规则组或部分审计字段，必须逐项列出
   2. 若日历当前仅完成月视图，也必须明确说明这是首期边界
