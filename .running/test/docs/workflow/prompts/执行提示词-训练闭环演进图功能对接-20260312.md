# 执行提示词-训练闭环演进图功能对接-20260312

请按照文件中的要求，在 `../workflow` 工作区把“训练闭环演进图”从 UI 示意升级为可执行、可追溯的真实功能对接。

## 1. 先阅读的文件
1. `docs/workflow/requirements/需求详情-统一入口与训练优化模块.md`（重点：`FR-UO-32/33/34`，`AC-UO-29/30/31/32`）

3. `docs/workflow/prototypes/统一入口与训练优化模块/训练闭环参考图.html`（仅作为视觉与布局对齐，不再作为数据示意来源）

## 2. 本次实现目标（必须）
在保持训练中心现有视觉风格、信息架构、固定窗口与区内滚动不变的前提下，完成以下功能对接：

1. 演进图与右侧节点详情的数据必须由后端接口返回驱动。
   1. 禁止默认使用前端硬编码示例节点目录（例如 `trainingLoopNodeCatalog()` 这类静态结构）。
   2. `include_test_data=false` 时，页面不得渲染“测试数据演进图”来冒充真实历史。
   3. 若历史不足：展示空态或最小结构，并明确标注 `暂无历史/暂无指标`。

2. “进入下一轮 / 回退本轮新增”必须可执行且可追溯。
   1. 点击按钮必须调用后端接口产生真实状态变化，并写入可追溯记录（建议复用 `training_audit_log`）。
   2. 动作需要幂等/防重复保护：重复触发不能造成不可控多次推进/回退。

3. 指标不可用时禁止伪数据。
   1. 若后端尚未接入真实评分/阈值计算，必须显式返回 `metrics_available=false`（或等效字段）与原因。
   2. 前端展示为 `暂无评分/暂无阈值`，禁止用固定示例数值默认填充。

## 3. 建议实现路径（可调整，但必须满足验收口径）
1. 增加后端接口：
   1. `GET /api/training/queue/{queue_task_id}/loop`（或等效）
   2. `POST /api/training/queue/{queue_task_id}/loop/enter-next-round`（或等效）
   3. `POST /api/training/queue/{queue_task_id}/loop/rollback-round-increment`（或等效）
2. 增加落库：
   1. 推荐新增 `training_loop + training_loop_node`（或 `training_loop_state`）表承载演进图状态。
   2. 关键动作写入 `training_audit_log`（action 建议为 `enter-next-round` / `rollback-round-increment`）。
3. 前端改造：
   1. 演进图与右侧面板改为消费 `GET /loop` 的返回数据渲染。
   2. 点击演进图节点切换右侧面板，节点 id 由后端返回。
   3. 点击动作按钮触发 `POST`，成功后刷新演进图与右侧面板。
