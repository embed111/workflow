## 3. 本轮目标
1. 本轮不是重做整个缺陷模块，而是在现有“排序 + 顺序建单”基础上补齐两条关键约束：
   1. `所有缺陷共用一个总开关 + 一个主动处理位 + 开启后串行建单` 的全局门禁；
   2. `task_priority` 的统一真相源；
   3. 同一缺陷动作的建单幂等与重复图收口。
2. 你需要完成：
   1. 实现修正；
   2. 历史数据/当前运行态收口策略；
   3. 可复验的验收证据回传。

## 4. 你必须按以下口径实现
1. 所有缺陷共用一个总开关、一个主动处理位和一张全局主图：
   1. 缺陷页头的“按顺序创建任务”是作用于所有缺陷的全局总开关，不是单条缺陷各自的开关；
   2. 总开关开启后，只有当前主动处理缺陷进入 `已解决` 或 `已关闭` 后，系统才允许为下一条缺陷建单；
   3. 在上一条缺陷仍处于 `未解决`、`有分歧` 或处理链未结束时，系统不得为下一条缺陷提前建单，也不得先在 `任务中心全局主图` 中预挂新的有效节点链；
   4. 所有缺陷任务统一挂到 `任务中心全局主图`，不要再把“每条缺陷各保留一张独立有效任务图”当成目标。
2. 用户显式优先级必须成为第一真相源：
   1. 若用户在摘要前缀、结构化正文或等效字段中已显式表达 `P0 / P1 / P2 / P3`，系统必须优先解析并写入 `task_priority`；
   2. 只有在用户未显式表达优先级时，才允许回退到启发式推断；
   3. 不允许继续出现“摘要写 `[P0]`，落盘 `task_priority=P1`”的分裂状态。
3. 同一条缺陷记录的以下对象必须使用同一个优先级值：
   1. 缺陷记录接口返回的 `task_priority`；
   2. 缺陷列表排序；
   3. 页头队列摘要中的 `active_task_priority / next_task_priority`；
   4. 自动或手动创建出的 `任务中心` 节点 `priority / priority_label`。
4. 对同一 `report_id + action_kind`，系统必须在唯一全局主图内收口为单条有效处理链：
   1. 同一缺陷的 `process` 动作在任意时刻最多只允许 1 条有效 `process` 链；
   2. 同一缺陷的 `review` 动作在任意时刻最多只允许 1 条有效 `review` 链；
   3. 重复点击、页面刷新、自动推进、重试都不得在 `任务中心全局主图` 中生成第二条并存有效链，也不得额外生成第二张有效任务图；
   4. 缺陷详情中的 `task_ref`、`ticket_id`、`external_request_id` 必须与 `任务中心全局主图` 中的有效节点链一一对应。
5. 若历史运行态中已经存在重复任务图：
   1. 必须给出明确的收口策略；
   2. 必须说明如何收口进唯一的 `任务中心全局主图`；
   3. 必须说明其余重复图如何处理：
      1. 清理；
      2. 归档；
      3. 隐藏；
      4. 标记失效。
   4. 不允许只修“以后不再生成”，却放任当前 8090 上的重复图继续并存无解释。
6. 本轮要继续保持之前已达成的行为不回退：
   1. “按顺序创建任务”条带、当前主动处理缺陷、下一条待建单缺陷仍需保留；
   2. `DTS-00003` 这类已能在 `任务中心` 查到的任务，不要回退成“缺陷详情有引用但任务中心不可见”；
   3. `有分歧` 记录仍属于可处理队列，不能被排除。

## 5. 明确禁止
1. 不要只改前端显示文案，而不修后端落盘和建单逻辑。
2. 不要继续依赖“摘要看起来像 P0”但实际接口仍返回 `P1` 的隐式推断。
3. 不要用“刷新后只显示一条”掩盖后端其实已经创建了多张任务图的事实。
4. 不要在没有说明清理策略的情况下直接删除现有历史数据。
5. 不要把当前已不再成立的旧问题继续当作主修复目标，例如“`asg-20260325-132002-d119da` 完全不显示在任务中心”。

## 6. 推荐实现顺序
1. 先定位并修正优先级来源链：
   1. 前端提交缺陷时是否传递显式优先级；
   2. 后端创建缺陷时如何解析并落盘 `task_priority`；
   3. 列表排序、队列摘要和建单桥接是否都复用同一字段。
2. 再定位并修正建单幂等链：
   1. `process` / `review` 动作的 `external_request_id` 生成与复用逻辑；
   2. 缺陷侧 `task_ref` 去重逻辑；
   3. `任务中心全局主图` 挂链是否还有绕过去重口径或绕过全局串行门禁的入口。
3. 最后处理历史数据：
   1. 检查 `dr-20260325-ca49dd19d9` 当前重复图；
   2. 给出收口进唯一全局主图后的保留/清理/归档方案；
   3. 用验收证据证明当前运行态已收口。
