# 专家治理目录

## 定位
- 本目录是 `workflow(pm)` 当前用于治理“小伙伴专家成长、理论库沉淀、问题-方法映射和学习策略替换”的真相源目录。
- 本目录不替代每个角色自己的工作区知识体；它负责全局治理、索引、准入/替换/淘汰规则，以及后续“自我修炼功能”的固化门槛。
- 当前阶段采用“文件治理优先、脚本辅助随后、产品功能最后固化”的推进方式。

## 当前结构
- `自我学习提升方法.md`
  - 当前全局学习策略文件。
  - 后续允许通过替换此文件整体升级学习方法，而不要求先改产品功能。
- `专业能力总表.md`
  - 全局角色能力总览。
  - 维护每个角色的专业方向、成熟度、核心能力项和下一阶段补强点。
- `问题-方法映射总表.md`
  - 遇到什么问题，默认优先用什么方法、交给谁、参考什么资产。
- `角色理论库建设清单.md`
  - 维护各角色理论库的目标结构、当前状态和首批建设清单。
- `templates/`
  - 方法卡、案例卡、外部资料评审卡模板。
- `material-reviews/`
  - 外部资料纳入、替换、合并、拒绝和淘汰评审记录。
  - 当前默认按 `material-reviews/YYYY-MM/README.md + YYYY-MM-DD-<role>-<topic>.md` 维护月度脚手架和具体评审卡。

## 治理规则
1. `自我学习提升方法.md` 是“怎么学习”的单一事实源，不在其他文件重复维护完整流程。
2. `专业能力总表.md` 只维护能力索引、成熟度和入口，不复制大段理论正文。
3. 理论库全文默认锚定在各角色自己的工作区；`pm/expertise/` 只维护治理入口、总表、映射和模板。
4. 外部资料不能直接进入理论库；必须先通过 `material-reviews/` 的评审卡，再决定 `纳入 / 合并 / 替换 / 拒绝 / 延后观察`。
5. 方法论资产默认允许被更优案例替换，也允许把过期、重复、弱案例降级或剔除。
6. 不收大段搬运资料；只保留能转成“问题 -> 方法 -> 步骤 -> 案例 -> 适用边界”的高信号内容。
7. `pm/expertise/material-reviews/YYYY-MM/README.md` 不再手工裸建。
   - 默认使用 `python .repository/pm-main/scripts/bin/refresh_pm_expertise_material_reviews.py --shell-root .` 补齐当前月目录和 README，再把具体评审卡写进对应月份。
8. 角色工作区 `state/role-assets/` 的结构化脚手架不再手工裸建。
   - 默认使用 `python .repository/pm-main/scripts/bin/refresh_pm_role_asset_scaffold.py --shell-root .` 补齐 `METHODS_INDEX.md` 与 `methods / cases / anti-patterns / reviews` 目录，再继续沉首批方法卡或案例卡。

## 与当前版本关系
- 当前目录直接服务 `V3`：
  - `V3-R1`：把学习提示和学习闭环收成稳定机制
  - `V3-R2`：把角色专业边界和能力责任收成稳定矩阵
  - `V3-R3`：把知识清理、替换和淘汰规则制度化
  - `V3-R5`：把测试资产 owner cadence、方法论和案例积累制度化
- 当前阶段不把“自我修炼功能”直接做成 UI/产品功能，而是先把文件治理跑稳。

## 推荐更新顺序
1. 先更新 `自我学习提升方法.md`，确认当前方法论入口和阶段门槛。
2. 再更新 `专业能力总表.md`，把角色成熟度和补强点同步追平。
3. 再更新 `问题-方法映射总表.md`，把高频问题映射到默认方法。
4. 若本轮引入外部资料，再写 `material-reviews/YYYY-MM/*.md`。
5. 若某个角色的方法卡、案例卡或反例卡要新增/替换/淘汰，再更新角色工作区里的对应理论库。
