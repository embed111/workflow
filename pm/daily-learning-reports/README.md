# 每日学习报告目录

## 定位
- 本目录用于保存每天每个小伙伴真实输出的学习报告。
- 学习报告不是 PM 的口头提示，也不是 PM 代写的空壳摘要。
- 默认要求每个对象都留下自己的产物，至少包括：
  - `workflow(pm)`
  - `workflow_devmate`
  - `workflow_testmate`
  - `workflow_qualitymate`
  - `workflow_bugmate`
  - `workflow_ucdmate`（若已进入正式协作口径）

## 路径规则
- 路径模式：`pm/daily-learning-reports/YYYY-MM-DD/<agent_id>.md`
- 一天内每个小伙伴只保留 `1` 份主学习报告；允许当天在同一文件中补充
- 文件存在不等于完成，内容必须带真实学习输出来源

## 最低内容
- `date`
- `agent_id`
- `learning_task`
- `source_type`
- `source_ref`
- `learned_points`
- `applied_to_project`
- `next_action`

## 真实性要求
- `source_type` 至少明确一种：
  - `helper_run`
  - `workspace_output`
  - `pm_self_learning`
- `source_ref` 必须可追溯到真实 run、真实文件或真实工作区产物
- PM 不得代写其他小伙伴的空壳学习报告；若当天没有真实输出，应该显式记为未完成，而不是补一句泛化总结
