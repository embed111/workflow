# 版本治理模板

## 定位
- 本目录提供版本治理的标准模板。
- 目标不是增加留痕数量，而是让 `需求拆分 -> 澄清 -> 方案评审 -> 测试设计 -> 演示 -> 测试 -> 发布收口` 这条链有稳定真相源。

## 模板清单
- `需求台账模板.md`
- `需求澄清记录模板.md`
- `方案设计评审模板.md`
- `测试设计评审模板.md`
- `功能演示验收模板.md`
- `阶段看板模板.md`
- `迭代甘特图模板.md`

## 使用规则
- 新版本激活前，至少从本目录复制并落地：
  - `需求台账模板.md`
  - `阶段看板模板.md`
  - `迭代甘特图模板.md`
- 某条需求进入 `clarified` 及以上阶段后，再按需创建：
  - 需求澄清记录
  - 方案设计评审
  - 测试设计评审
  - 功能演示验收
- 热修、gate 补洞和兼容性修补允许走快车道，但仍要保留一份合并记录，不得跳过事实留痕。
