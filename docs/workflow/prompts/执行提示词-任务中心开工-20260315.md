## 2. 本次任务性质
1. 本次是 `任务中心` 的正式实现开工，不是继续补示意原型。
2. 需求与原型口径已经同步，必须以当前交付文件为准。
3. 本次按第一版直接实现，不做历史方案迁移描述。

## 3. 本次必须完成的范围
1. 左侧导航新增并接入 `任务中心` 模块，口径固定为：
   1. `对话`
   2. `工作记录与训练`
   3. `训练中心`
   4. `任务中心`
   5. `设置`
2. 页面主结构按原型实现：
   1. 页头调度状态与动作区
   2. 中部任务图
   3. 右侧任务详情
3. 页头必须实现：
   1. 全局状态：`运行中 / 暂停中 / 已暂停 / 未启动`
   2. 文案：`并行agent数`
   3. 动作：`暂停调度 / 继续运行`
4. 创建任务必须实现为抽屉表单，而不是独立页面。
5. 当前创建任务表单字段固定为：
   1. `任务名称`
   2. `执行 agent` 下拉
   3. `优先级` 下拉
   4. `确认任务目标`
   5. `预期产物（选填）`
   6. `上游任务（可选，支持搜索添加）`
6. 当前创建任务表单明确不做：
   1. `下游任务` 直填
   2. `提交说明`
   3. 标题下的大段解释文案
7. 任务图必须实现当前已确认的核心规则：
   1. 已执行任务在未执行任务之上
   2. 阻塞任务属于未执行任务
   3. 节点名称固定在图形右侧
   4. 左侧独立时间列展示完成时间
   5. 历史任务支持分批加载
8. 任务详情必须实现：
   1. 基础摘要
   2. 回执展示
   3. 失败任务动作：`重跑任务`、`人工修改执行状态`
9. 后端必须补齐当前首期最小闭环：
   1. 任务图创建/读取
   2. 单节点创建
   3. 依赖边保存与反向下游读取
   4. 调度状态读取
   5. `dispatch-next`
   6. `pause / resume`
   7. `mark-success / mark-failed`
   8. `rerun / override-status`
10. 优先级必须保持前后端一致：
   1. 页面展示：`P0/P1/P2/P3`
   2. 后端归一化：`0/1/2/3`

## 4. 明确禁止
1. 不要只做静态前端 mock。
2. 不要把当前创建任务表单改成“上下游都直填”。
3. 不要额外扩出独立创建页或顶部说明堆叠方案。
4. 不要脱离现有 `workflow` 视觉语言另起一套看板风格。
5. 不要遗漏后端依赖门禁、单 agent 单并发和全局并发上限规则。

## 5. 推荐实现顺序
1. 先接入左侧导航和 `任务中心` 页面路由。
2. 再落任务图主结构和右侧详情联动。
3. 再落创建任务抽屉表单。
4. 再补后端接口与数据落盘。
5. 最后联调：
   1. 创建任务
   2. 任务图刷新
   3. 暂停/继续运行
   4. 失败重跑/人工改状态
