# 执行提示词-训练闭环入口统一与中栏滚动修正-20260313

请按照以下文件中的最新口径执行本次训练闭环界面收口，不要回退到旧版三列创建页或顶部 `创建任务 / 任务状态` 切换：

## 1. 先读文件
1. `docs/workflow/requirements/需求详情-训练闭环页签与滚动分层约束.md`
2. `docs/workflow/design/详细设计-统一入口与训练优化模块.md`
3. `docs/workflow/prototypes/统一入口与训练优化模块/训练闭环-创建任务参考图.html`
4. `docs/workflow/prototypes/统一入口与训练优化模块/训练闭环参考图.html`

## 2. 本次实现范围
1. 训练闭环左侧任务列表收口为唯一主入口：
   1. `+ 新建训练任务` 进入创建态；
   2. 选择已有任务进入对应任务状态页；
   3. 顶部不再保留与之重复的 `创建任务 / 任务状态` 模式切换。
2. 创建页改为“两列”：
   1. 左侧沿用与状态页一致的任务列表骨架；
   2. 中部承载创建表单、页签、训练路径预览与提交区；
   3. 删除右侧 `创建结果与校验` 重复栏目。
3. 创建页提交动作收口：
   1. 左侧 `+ 新建训练任务` 仅表示进入创建态；
   2. 真正提交动作只保留 `保存并启动首轮 / 保存草稿`。
4. 最宽中栏改为统一主滚动容器：
   1. 创建页中栏整体滚动；
   2. 状态页中栏整体滚动；
   3. 不允许出现“已有滚动条但内容仍被压缩/裁切”的视觉 bug。
5. 中栏内部已有页签必须是真实可切换，不允许只做静态示意：
   1. 创建页：`基础信息 / 首轮工作集 / 启动确认`
   2. 状态页：`当前概览 / 工作集变化 / 三轮评测 / 历史记录`

## 3. 明确禁止
1. 不要恢复顶部 `创建任务 / 任务状态` 模式切换。
2. 不要恢复创建页右侧 `创建结果与校验` 栏。
3. 不要把“有滚动条”误当作“内容完整可见”；必须处理内容被 `flex-shrink` 压缩或被内层容器裁切的问题。
4. 不要让左侧任务列表在创建页和状态页演化成两套不同骨架。

## 4. 最低验收要求
1. 视觉验收：
   1. 创建页与状态页左侧任务列表骨架一致；
   2. 创建页为两列，状态页为三列；
   3. 中栏底部内容可通过滚动完整查看，不出现半截卡片或底部被截断。
2. 交互验收：
   1. 中部页签点击后真实切换内容；
   2. 切换页签后中栏滚动位置回到顶部；
   3. 不出现整页长滚动替代区内滚动。
3. 证据回传：
   1. 创建页截图：左栏统一骨架 + 中栏提交区 + 中栏滚动到底部；
   2. 状态页截图：左栏统一骨架 + 中栏滚动到底部；
   3. 页签切换截图：至少 1 张创建页、1 张状态页；
   4. 若改动了前端脚本，请附对应代码路径。
