请在 `../workflow` 按以下要求实现并回传证据。

## 1. 输入文档（以本工作区 `docs/workflow/` 为准）
1. `docs/workflow/requirements/需求详情-角色画像发布格式与预发布判定.md`

3. `docs/workflow/overview/需求概述.md`（第 12 节增量口径）

## 2. 必做实现
1. 角色画像展示形态固定为“单栏阅读流（A）”，不使用双栏并列主体区。
2. 详情主体顺序按文档执行：
   - `身份头卡 -> 我能做什么 -> 知识范围 -> 核心技能 -> 适用场景 -> 版本说明 -> 发布历史与操作`
3. 支持角色头像本地上传：
   - 上传入口可见；
   - 成功后立即生效；
   - 失败回退默认头像并提示原因。
4. 头像上传限制强约束：
   - 格式仅 `png/jpg/webp`；
   - 单文件 `<=2MB`。
5. 保持既有发布/预发布语义门禁：
   - 发布字段校验；
   - 不合规降级普通提交；
   - `git status --porcelain --untracked-files=normal` 预发布判定。

## 3. 禁止项
1. 禁止引入双栏摘要卡作为主体画像布局。
2. 禁止绕过头像格式/大小校验直接写入。
3. 禁止仅修改文案不落地交互与校验逻辑。

## 4. 验收与回传
1. 严格对齐 `AC-RP-01 ~ AC-RP-09`。
2. 每条 AC 至少回传：
   - 功能截图（非统计截图）；
   - API 请求/响应或日志证据；
   - 代码路径。
3. 额外回传：
   - 头像上传成功与失败各 1 组证据；
   - 单栏布局首屏截图（可证明无双栏并列主体区）。
