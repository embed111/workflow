## 2. 本轮必须完成
1. `scripts/` 分层整理（必须）：
   1. `scripts/bin/`：兼容入口脚本（对外命令）。
   2. `scripts/quality/`：门禁与检查脚本（`check-*`）。
   3. `scripts/dev/`：开发/启动辅助脚本。
   4. `scripts/acceptance/` 保持现有职责不变。
2. 顶层瘦身：
   1. `scripts/` 顶层只保留最小入口集合（推荐 <= 4 个）。
   2. 顶层剩余文件必须是“稳定入口或迁移过渡 stub”，禁止业务脚本散落。
3. 兼容性要求：
   1. 旧命令路径仍可用（通过 stub 或转发）。
   2. `run_workflow.bat`、现有 README 命令示例不得失效。
4. 文档更新：
   1. 新增 `scripts/README.md`，提供“旧路径 -> 新路径”映射表。
   2. 更新 `docs/workflow/reports/ARCHITECTURE_REFACTOR_MAP.md` 与 `docs/workflow/reports/REFactor_LINE_BUDGET_REPORT.md` 记录本轮目录治理。

## 3. 第八轮硬门禁（任一失败即 `fail_hard_gate`）
1. 顶层门禁：
   1. `scripts/` 顶层文件数 `<= 4`。
2. 分类门禁：
   1. 所有 `check-*.py` 必须位于 `scripts/quality/`（顶层仅可留同名兼容 stub）。
   2. 启动脚本（`.ps1`）必须位于 `scripts/dev/`（顶层仅可留兼容 stub）。
3. 兼容门禁：
   1. 旧路径命令执行成功（至少抽测 5 条历史命令）。
4. 回归门禁：
   1. UO + AR 脚本均 `exit code 0`。
5. 质量门禁：
   1. `scripts/check-module-readme-contract.py`（或新路径等效入口）通过。
   2. `scripts/check-prototype-text.py`（或新路径等效入口）通过。

## 4. 禁止事项
1. 禁止删除对外入口而不提供兼容转发。
2. 禁止把脚本仅改名不改分类。
3. 禁止夹带新功能需求。
4. 禁止只回传门禁统计截图，缺少功能截图/API/日志/代码路径。
