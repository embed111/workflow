## 2. 2026-03-31 10:08~10:16 +08:00 live 已核实结论
1. 这次问题不只是“训练优化没铺满屏幕”，而是更上层的模块显示链路就已经错了。
2. 当前 live 版本：`20260331-203853`。
3. live 取证结果：
   1. 在 `角色资产与版本` 处于激活态时，`#tcModuleOps` 虽然 class 还是 `tc-module`，但 `getComputedStyle(...).display = "grid"`，没有被隐藏。
   2. 结果是 `训练优化` 整块被直接挂在 `角色资产与版本` 页面下方一起渲染，整页被拉成长文档。
   3. 资产页状态下，`appRoot` 高度约 `4632px`。
   4. 进入 `workflow_tester -> 训练优化` active 态后，`appRoot` 高度仍约 `3442px`，并不是固定单窗口工作台。
   5. active 态时可观察到：
      1. `bodyScrollTop ≈ 2738`
      2. `appRoot.top ≈ -2738`
      3. `tcModuleOps.top ≈ -2438`
      这说明当前页面已经变成 body 级长页滚动，而不是工作台内部滚动。
4. 当前 root cause 已定位：
   1. `index.html` 中 `#tcModuleOps` 本应只是普通 `tc-module`，显示应由 `.tc-module.active` 决定；
   2. 但 `index_release_review.css` 中直接写了：
      1. `#tcModuleOps { display: grid; ... }`
   3. 这条规则覆盖了通用 `tc-module` 的隐藏语义；
   4. 后续 `index_training_loop_overview.css` 又继续给 `#tcModuleOps .tc-loop-grid` 注入完整三栏布局，所以问题被放大成整页超长。
5. 次级结论：
   1. 我这轮调试窗口视口只有约 `754x487`；
   2. 已命中 `@media (max-width: 980px)` 的单栏断点；
   3. 这会让训练优化在小窗口下退成单栏，但这不是本轮“工作台没铺满 / 页面变长页”的主因。
6. 当前 `工作区升级` banner 已改到顶部居中，本轮不把它当成首因，不要继续误判。

## 3. 本轮目标
1. 先恢复 `训练优化` 的模块显示语义：
   1. 未激活时不参与页面布局；
   2. 激活后独占当前主工作台区域。
2. 让 `训练优化` 恢复为固定单窗口工作台，而不是 body 长文档滚动页。
3. 在此基础上，再保证它在桌面窗口下真正铺满主工作台，而不是内部被裁切、被滚走或首屏不稳定。

## 4. 本轮必须整改到位的点
1. 修正 `#tcModuleOps` 的隐藏 / 激活语义：
   1. 未激活时必须真正 `display:none` 或等效不参与布局；
   2. 不能继续因为某条高优先级 CSS 规则而强行显示。
2. 修正 `角色中心` 模块切换后的高度链路：
   1. `pane-training-center`
   2. `.training-center-wrap`
   3. `#tcModuleOps`
   4. `.tc-loop-grid`
   5. `tc-loop-side / center / right`
   这些层级必须重新形成稳定的 `100% / min-height:0 / overflow` 传递链。
3. active 进入 `训练优化` 后必须满足：
   1. 主工作台首屏稳定；
   2. 不出现 body 级长页滚动；
   3. 需要滚动的内容只在工作台内部滚动容器里滚。
4. 不要把“铺满屏幕”误做成“内部卡片更宽一点”：
   1. 根问题不是先调 `grid-template-columns`；
   2. 根问题是模块显示冲突和滚动层级错位。
5. 响应式断点仍要保留，但语义要正确：
   1. 小窗口下允许变单栏；
   2. 但单栏也必须仍是固定工作台内部布局；
   3. 不能退化为整页长文档。

## 5. 推荐实现顺序
1. 先修 `#tcModuleOps` 被强制显示的问题：
   1. 把 `display:grid` 从模块根节点移走；
   2. 只在 `active` 或内部容器上定义布局。
2. 再检查 `角色中心` 顶层高度链：
   1. `.app`
   2. `.workspace`
   3. `#pane-training-center`
   4. `.training-center-wrap`
   5. `#tcModuleOps`
3. 再修内部滚动层：
   1. 左侧会话列表
   2. 中部聊天壳
   3. 右侧任务 / 能力演进
   4. 右侧执行结果
4. 最后再调桌面三栏与小屏单栏的列宽、gap、断点。
