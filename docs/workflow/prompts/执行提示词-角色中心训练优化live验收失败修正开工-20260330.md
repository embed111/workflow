   1. `基础信息`
   2. `首轮工作集`

## 3. 本轮目标
1. 把 `训练中心` 的主产品口径真正收口为 `角色中心（原训练中心）`。
2. 把 `训练闭环` 子模块真正改成 `训练优化`，并按需求切换为与 `创建角色` 同类的对话式工作流。
3. 即使当前没有任何训练队列数据，也要让 `训练优化` 首屏展示正确的新结构：
   1. 左侧优化会话列表或等效空态列表
   2. 中部单聊天壳
   3. 中部聊天壳中的能力列表区域或能力列表空态
   4. 右侧默认 `任务 / 能力演进`
   5. 右侧可切换 `当前能力基线`
4. 让后续二次验收不再卡在“页面还是旧入口、旧布局、旧术语”这一层。

## 4. 你必须按以下口径修正
1. 命名与导航收口：
   1. 页面主称呼改为 `角色中心`；
   2. 子模块主称呼改为 `训练优化`；
   3. 不要继续在主导航、主标题、主按钮中保留 `训练闭环` 作为当前有效主称呼。
2. 训练优化主路径修正：
   1. `训练优化` 的默认主路径必须是对话式工作流；
   2. 不允许继续让旧的三步表单 `基础信息 / 首轮工作集 / 启动确认` 占据默认首屏；
   3. 如果出于兼容原因还要保留旧训练创建器，只能作为次级动作或历史兼容入口，不能作为进入 `训练优化` 后的默认主界面。
3. 空态结构必须正确：
   1. 即使当前没有任何训练优化会话、没有任何任务、没有任何能力对象，也必须展示：
      1. 中部聊天壳空态
      2. 中部能力列表空态
      3. 右侧 `任务 / 能力演进` 默认标签
      4. 右侧 `当前能力基线` 第二标签
   2. 不要因为 `queue=0` 就回退为旧表单创建页。
4. 能力列表与门禁绑定：
   1. 当存在能力对象数据时，中部聊天壳内必须直接展示能力列表；
   2. 每条能力项必须同时具备：
      1. `能力展示效果`
      2. `能力评分`
      3. `Gate-B`
      4. `Gate-C`
      5. `历史能力影响`
   3. 不允许只显示会话级总门禁结论，而看不到具体能力项的绑定结果。
5. 右侧辅助区：
   1. 默认显示 `任务 / 能力演进`；
   2. 第二标签显示 `当前能力基线`；
   3. 即使空态，也要保留这两类容器和可理解标题，不要整块缺失。
6. 旧入口兼容边界：
   1. 允许保留“新建训练任务”这类动作按钮；
   2. 但点击前的主工作台必须已经是新口径；
   3. 不要用“按钮存在”替代“主结构已完成”。

## 5. 明确禁止
1. 不要只改文案，把 `训练闭环` 改成 `训练优化`，但主体仍是旧三步表单。
2. 不要只在有数据时才渲染新结构，空态时又回退旧布局。
3. 不要把能力列表挪到右侧，导致中部聊天壳仍然只有普通消息或说明文字。
4. 不要继续让 `训练优化` 缺少右侧 `任务 / 能力演进` 与 `当前能力基线` 双标签结构。
5. 不要为了过验收而只补静态壳，但没有能力对象字段或门禁字段落点。

## 6. 推荐实现顺序
1. 先修命名、导航、模块路由和默认主结构。
2. 再补 `训练优化` 空态壳：
   1. 左侧会话列表空态
   2. 中部聊天壳空态

   4. 右侧双标签空态
