## 2. 本次实现目标
请在保持当前 `workflow` 训练中心视觉风格、信息架构和客户端式固定窗口约束不变的前提下，实现以下调整：

1. 训练闭环改为“流程中的当前一步”展示，而不是只靠若干结果卡和详情块拼出流程感。
   1. `创建任务` 与 `任务状态` 两个模式共用同一套五阶段语义：
      1. `创建任务`
      2. `配置本轮工作集`
      3. `执行三轮评测`
      4. `生成结果判定`
      5. `进入下一轮 / 完成训练`
   2. 顶部主卡应显式区分：
      1. 已完成阶段
      2. 当前阶段
      3. 待进入阶段

2. `任务状态` 模式增加训练演进图，类似 `git graph`。
   1. 该图不是普通时间线，而是训练分叉、回退和收敛路径图。
   2. 至少能看出：
      1. 基线节点
      2. 劣化分支
      3. 回退节点
      4. 保留主线
      5. 当前活动节点
      6. 未来收敛目标
   3. 必须显式表达两类关键场景：
      1. “第一轮能力劣化 -> 回退本轮新增”
      2. “第一轮分数提升但仍不足阈值 -> 保留主线进入第二轮”

3. `任务状态` 右侧栏改为“当前选中演进节点详情面板”。
   1. 默认展示当前活动节点（示意为 `R2 当前`）的详情。
   2. 右侧至少展示：
      1. 节点名称
      2. 所属轮次
      3. 节点得分/关键指标
      4. 节点判定
      5. 下一步动作
      6. 与该节点直接相关的回退/保留说明
   3. 如果实现中支持点击演进图节点，右侧内容应联动切换；若暂未支持点击，也需先按当前活动节点静态对齐。

4. `创建任务` 模式保留表单主区，但也要使用同一套流程骨架。
   1. 顶部主卡改为当前阶段状态 + 五阶段流。
   2. 补充“训练路径预览”，提前说明：
      1. 首轮若劣化将回退
      2. 首轮若提升但不足阈值将进入下一轮
   3. 不要把创建态做成完全脱离运行态语言的另一套页面。

## 3. 视觉与交互约束
1. 保持现有训练中心的浅色工作台风格、边框语义和 `tc-*` 体系，不引入新的设计语言。
2. 固定窗口、底边平齐、区内滚动条、客户端式局部滚动必须保留。
3. 不允许退化为整页长滚动承载核心训练内容。
4. `任务状态详情` 继续保持：
   1. 带外框独立滚动区
   2. 二级标签切换
   3. 三级折叠承载长明细
5. 页面常驻区不要再放“如何阅读页面”“当前口径说明”这类设计解释文案。
6. 流程图和演进图应优先用结构表达语义，不要依赖大段说明文案。

## 4. 优先复用与实现提示
1. 优先复用现有训练中心布局、面板、按钮、标签和滚动容器。
2. 不要为流程图和演进图单独引入重型图库；优先使用现有 DOM/CSS/SVG 能力实现。
3. 训练演进图优先表达“关键节点 + 主线保留 + 劣化回退”，不要为了完整历史把图无限拉长。
4. 右侧节点详情面板只承载当前节点结论，不要把完整 run 表和长清单塞进去。
5. 完整评分明细、工作集变化、候选资料和迭代记录继续留在中部详情区。
