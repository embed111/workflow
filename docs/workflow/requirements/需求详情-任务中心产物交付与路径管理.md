# 需求详情-任务中心产物交付与路径管理

## 1. 主题目标与价值说明
1. 为 `任务中心` 增加统一的产物交付规则，避免各 agent 各自写文件、无法追踪。
2. 通过统一的产物根路径配置，确保所有任务产物都按固定目录结构落盘。
3. 通过“无交付人 / 指定交付人”两种情况，支撑默认落盘和点对点交付两类场景。
4. 将“任务成功”与“产物已交付”绑定，避免任务状态成功但产物不可追溯。

## 2. 用户画像与使用场景
1. 用户画像 A：workflow 操作者
   1. 关注点：创建任务时就明确产物交付方式，任务结束后能看到实际落盘路径。
2. 用户画像 B：平台治理者
   1. 关注点：所有任务产物是否都受统一根路径约束，交付链路是否可追溯。
3. 用户画像 C：接收产物的目标 agent
   1. 关注点：当任务指定自己为交付对象时，能否在统一约定目录下接收产物。
4. 核心场景：
   1. 管理员在 `设置` 既有内容中新增并维护产物根路径。
   2. 用户在 `任务中心` 创建任务时，选择产物交付类型。
   3. 若选择 `指定交付人`，则为该任务指定唯一交付目标 agent。
   4. 任务执行完成后，通过统一接口提交产物。
   5. 用户在任务详情中查看产物状态、交付类型和实际落盘路径。
   6. 未完成产物交付时，任务不能被人工修改为成功态。

## 3. 用户旅程或关键流程
1. 管理员进入 `设置`，在既有设置内容中配置 `产物根路径`。
2. 系统校验路径可用后，将其标记为当前生效路径。
3. 系统在产物根路径下创建 `workspace/` 目录，作为任务记录统一落盘位置。
4. 用户进入 `任务中心`，点击 `+ 创建任务`。
5. 在现有任务表单基础上补充：
   1. `产物交付类型`：`无交付人 / 指定交付人`
   2. `交付给 agent`：仅当选择 `指定交付人` 时显示，且单选
6. 任务保存后，系统为任务记录产物交付规则，并将任务记录写入 `workspace/` 目录。
7. 任务执行过程中，通过统一接口提交产物文件。
8. 系统按规则落盘：
   1. 无交付人：`<根路径>/<工作agent>/product/<任务名>/...`
   2. 指定交付人：
      1. `<根路径>/<工作agent>/product/<任务名>/...`
      2. `<根路径>/<交付agent>/receive/<任务名>/...`
   3. 任务记录：`<根路径>/workspace/...`
9. 用户在任务详情中查看：
   1. 产物交付状态
   2. 交付类型
   3. 交付对象
   4. 实际落盘路径
10. 若任务尚未完成产物交付，则“人工修改执行状态 -> 成功”动作被阻止。

## 4. 功能需求清单
1. FR-PM-01 产物根路径配置
   1. `设置` 中必须新增 `产物根路径` 配置区块。
   2. 该区块必须作为既有设置页中的新增内容出现，不得替代或遮蔽原有设置内容。
   3. 区块至少包含：
      1. 当前生效路径
      2. 选择本地路径入口
      3. 路径校验状态
      4. 任务记录目录：`<产物根路径>/workspace/`
   4. 系统必须保证所有任务产物都受该根路径约束。
2. FR-PM-02 不新增独立主模块
   1. 第一版前端不得新增左侧独立 `产物管理` 顶层模块。
   2. 产物能力在界面上仅通过 `设置` 与 `任务中心` 暴露。
3. FR-PM-03 创建任务表单增量字段
   1. `任务中心` 创建任务表单必须新增：
      1. `产物交付类型`
      2. `交付给 agent`
   2. `产物交付类型` 必须支持：
      1. `无交付人`
      2. `指定交付人`
   3. `交付给 agent` 仅在 `指定交付人` 时显示。
4. FR-PM-04 单交付人限制
   1. 第一版 `交付给 agent` 仅允许单选。
   2. 第一版不支持多个接收 agent 协作交付。
5. FR-PM-05 任务详情产物区块
   1. `任务中心` 右侧任务详情中必须新增 `产物` 区块。
   2. 该区块至少展示：
      1. 产物状态
      2. 交付类型
      3. 交付对象
      4. 实际落盘路径
      5. 最近一次交付时间
   3. 至少提供 `查看产物` 入口。
6. FR-PM-06 无交付人路径规则
   1. 当任务处于 `无交付人` 情况时，产物必须落到：
      1. `<产物根路径>/<工作agent>/product/<任务名>/...`
7. FR-PM-07 指定交付人路径规则
   1. 当任务指定交付人时，系统必须同时生成两份产物落盘：
      1. `<产物根路径>/<工作agent>/product/<任务名>/...`
      2. `<产物根路径>/<交付agent>/receive/<任务名>/...`
   2. 第二条路径中的 `交付agent` 必须取自任务创建时指定的接收对象。
8. FR-PM-08 成功门禁绑定产物交付
   1. 若任务未完成产物交付，则不得人工修改为成功态。
   2. 若系统尚未配置有效 `产物根路径`，则不得允许任务被人工修改为成功态。
9. FR-PM-09 统一交付接口
   1. 所有 agent 任务产物必须通过统一接口交付，不得绕过系统协议直接记成功。
   2. 统一接口至少需要记录：
      1. 任务标识
      2. 工作 agent
      3. 交付类型
      4. 交付对象（若有）
      5. 实际落盘路径
      6. 交付时间
10. FR-PM-10 路径预览
   1. 创建任务表单中应能预览当前任务的目标落盘路径。
   2. 任务详情中应展示最终实际落盘路径，而不是只展示抽象状态。
11. FR-PM-11 任务记录目录规则
   1. 系统必须在 `产物根路径` 下固定创建 `workspace/` 目录。
   2. 所有创建的任务记录必须统一存放到 `workspace/` 目录下。
   3. 推断/假设（置信度: 中）：`workspace/` 目录下可按任务图或任务标识进一步分层，但首期至少要保证根目录固定为 `<产物根路径>/workspace/`。

## 5. 非功能需求
1. 一致性：
   1. 页面展示的路径规则必须与后端真实落盘规则一致。
2. 可追溯性：
   1. 任一任务产物都应能追溯到任务、工作 agent、交付类型和接收对象。
   2. 任一任务记录都应能追溯到统一的 `workspace/` 目录位置。
3. 可用性：
   1. 用户在查看任务详情时，应能在 3 秒内判断“是否已交付、交付给谁、文件在哪”。
4. 安全性：
   1. 产物落盘路径必须限制在配置的产物根路径下，不得任意越界写入。

## 6. 验收标准
1. AC-PM-01 设置中配置产物根路径
   1. Given 用户进入 `设置`
   2. When 查看产物路径配置区
   3. Then 可看到既有设置内容仍保留
   4. And 可看到当前生效路径、选择本地路径入口与路径校验状态
   5. And 可看到任务记录目录为 `<根路径>/workspace/`
2. AC-PM-02 创建任务表单出现产物交付字段
   1. Given 用户点击 `+ 创建任务`
   2. When 表单展开
   3. Then 可看到 `产物交付类型`
   4. And `产物交付类型` 仅包含 `无交付人 / 指定交付人`
   5. And 选择 `指定交付人` 后可看到 `交付给 agent`
3. AC-PM-03 无交付人路径正确
   1. Given 某任务选择 `无交付人`
   2. When 任务提交产物
   3. Then 产物落到 `<根路径>/<工作agent>/product/<任务名>/...`
4. AC-PM-04 指定交付人路径正确
   1. Given 某任务选择 `指定交付人`
   2. And 交付对象为某单一 agent
   3. When 任务提交产物
   4. Then 工作 agent 本地目录保留一份
   5. And 接收 agent 的 `receive/<任务名>` 目录也收到一份
5. AC-PM-05 未交付产物不得改为成功态
   1. Given 某任务尚未完成产物交付
   2. When 用户尝试通过 `人工修改执行状态` 将任务改为成功
   3. Then 系统拒绝并提示需先完成产物交付
6. AC-PM-06 任务详情可见实际路径
   1. Given 某任务已完成产物交付
   2. When 用户打开该任务详情
   3. Then 可看到交付类型、交付对象、实际落盘路径与最近一次交付时间
7. AC-PM-07 第一版不出现独立产物主模块
   1. Given 用户查看左侧导航
   2. When 页面加载完成
   3. Then 不出现独立 `产物管理` 顶层模块
8. AC-PM-08 任务记录目录固定
   1. Given 用户已配置产物根路径并创建任务
   2. When 系统写入任务记录
   3. Then 任务记录被统一写入 `<根路径>/workspace/` 目录下

## 7. 边界条件与异常处理
1. 若未配置产物根路径：
   1. 任务仍可创建，但不得人工修改为成功态。
2. 若用户选择 `指定交付人` 但未选择接收 agent：
   1. 系统必须拒绝保存该任务。
3. 若交付接口写入路径超出产物根路径：
   1. 系统必须拒绝写入，并返回路径越界错误。
4. 若任务名称包含不适合作为目录名的字符：
   1. 系统必须进行安全归一化处理，并在展示层保持可读映射。
5. 若指定交付人不存在于当前角色池：
   1. 系统必须拒绝保存该任务。
6. 若 `workspace/` 目录创建失败：
   1. 系统必须拒绝将任务创建标记为成功，并返回明确错误。
